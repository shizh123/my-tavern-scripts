# 秦璐重置版 — 去时间化总原则（楼层驱动一切）

> 状态：基调已敲定，逐项改造方案待细化。
> 记录时间：2026-06-24
> 关联：《念头培育与时间系统.md》《心防松动窗口.md》《商店道具系统.md》

---

## 〇、总原则（最高基调）

**游戏时间（日期/时间/小时/天）不再参与任何计算逻辑，全部改为楼层（对话轮数）驱动。**

- 时间对这个游戏**不再重要**，日期/时间只作为**氛围显示**保留（让场景有昼夜感），不驱动任何系统。
- 所有"以小时/天为单位的阈值、冷却、时效、过期、衰减"一律改为"以楼层/轮数为单位"。
- 所有"结束时间戳"改为"到期楼层"或"剩余轮数"。
- **单一楼层基准**：全系统共用同一个"当前楼层"来源（建议统一封装一个 `getCurrentFloor()`），念头培育、心防松动窗口、道具时效、危险累计、疑心衰减全部以它为准，杜绝多套计数。

### 无例外：苏文位置也改楼层驱动

**去时间化无任何例外。连苏文位置都改为楼层驱动，时间完全退出所有计算。**

- 苏文位置 = 楼层驱动的黑盒生活节律：楼层即时间流逝，位置随楼层变化。
- 映射原理黑盒，不告知玩家、不告知 AI，对外只暴露当前位置。
- "星期"概念保留，但由楼层推算，不靠真实时间。
- 详见《苏文系统.md》§四。

> 历史注记：曾一度考虑"时间段专供苏文作息"作为唯一例外，后**推翻**——苏文位置改楼层驱动，去时间化彻底无例外。

> 这条原则覆盖旧版几乎所有核心系统，是重置版的地基。下文逐系统列出"旧用时间做什么 → 改成楼层怎么做"。

---

## 一、时间推进引擎（源头）

| 旧版 | 改造方案 |
|------|---------|
| 每回合自动 `+1 小时`；玩家输入含"一周后/几小时后"则跳跃 N 小时（`advanceTime` / `detectTimeJump` / `TIME_JUMP_RULES`） | **取消每回合 +1 小时**。每回合 = 楼层 +1，仅此而已。 |
| `isTimeJump`（跳跃>1小时）作为很多分支的开关 | **废弃时间跳跃驱动逻辑**。"第二天/一周后"只作为**叙事氛围**由 AI 自由叙述，不再换算成系统跳跃，不影响任何计数。 |
| `_推进前时间` 作为习惯"形成时间"时间戳 | 改记 **形成楼层**（或保留一个时间戳仅作展示）。 |
| 日期 → 计算星期 `世界.星期` | 可保留为纯显示；不再驱动苏文作息（见§十）。 |

> 时间显示（`世界.时间/日期`）由 AI 按剧情自由维护：剧情跨天时更新日期，时间随叙事自然变化；脚本不做粗粒度推进，也不参与任何计算。

## 二、念头培育 / 成熟（已在《念头培育与时间系统.md》定调）

| 旧版 | 改造方案 |
|------|---------|
| 念头 `需要时间`（3-5 小时）、`开发进度` 累积小时；`getBaseHours(stage)` 返回小时 | 改为 **需要楼层数**（如简单 3 楼 / 困难 6 楼，数值待平衡）。 |
| 进度推进绑游戏时间；时间跳跃 +1 且过期延长 +12h | 改为**每楼 +1 保底**，AI 相关性加速叠加（见培育文档 §3）。 |
| 成熟判定 `开发进度 >= 需要时间`（小时） | 语义改为 `开发进度(楼) >= 需要楼层数`，逻辑不变、单位改。 |

> 详见《念头培育与时间系统.md》，本条只作去时间化归口。

## 三、念头过期

| 旧版 | 改造方案 |
|------|---------|
| 植入时 `过期时间 = 植入时间 + 24h`；`isThoughtExpired` 比日期 | 改为 **过期楼层 = 植入楼层 + N 楼**（N 待定），`当前楼层 >= 过期楼层` 即过期。 |
| 被拒念头用 `需要时间=26`（>24h 必过期）实现"软拒绝" | 重置版判定改为**枚举状态机**（待判定/培育中/已成熟/已过期），不再用"26 小时"魔法数字表示拒绝（对齐《念头判定机制.md》§四）。 |
| 待判定念头 `植入时间 + 2h` 超时自动定型 | 改为 **植入楼层 + N 楼**仍未判 → 处理（重置版倾向"保持待判定下一轮重试，不降级"，见判定文档）。 |

## 四、危险累计与苏文打断阈值

| 旧版 | 改造方案 |
|------|---------|
| `危险时间累计 += 1`（语义是小时，但实为每回合，时间跳跃只 +1 = 已知 bug） | 正名为 **危险楼层累计**，每满足危险条件的楼层 +1，bug 自然消失。 |
| 累计 `>=2 小时` 疑心+5、`>=4 小时` 疑心+10+触发打断（`INTERRUPTION_THRESHOLD=4`） | 阈值改为 **楼层数**（如 >=2 楼提醒、>=4 楼打断，数值待平衡）。 |
| 前端 `INTERRUPTION_THRESHOLD=4` 与脚本各一份 | **单一真相源**：阈值只定义一处，前端引用，不重复定义。 |

## 五、苏文巡逻：冷却与概率

| 旧版 | 改造方案 |
|------|---------|
| `cooldownHours=2`、`isPatrolCooldownOver` 算小时差 | 改为 **冷却楼层数**（如 2 楼），`当前楼层 - 上次巡逻楼层 >= N`。 |
| 概率按**小时段**修正（morning/afternoon/evening/night × 0.3~1.2） | 时间段不再驱动概率。改为：基础概率 × 疑心修正（去掉时段因子），或改用确定性种子（`seededRandom(楼层, 盐)`）替代裸 `Math.random()`，避免 ROLL 刷概率。 |
| `离开期限 = 当前 + 1~2 小时`，超时判定比时间 | 改为 **离开期限楼层 = 当前楼层 + N 楼**，`当前楼层 >= 期限楼层` 触发。 |
| 前端复算巡逻概率（与脚本重复） | 单一真相源，前端只读不复算（或复用同一纯函数）。 |

## 六、疑心冻结（借口成功 / 道具）

| 旧版 | 改造方案 |
|------|---------|
| 借口成功 `冻结结束时间 = 当前 + 2h`；到期比时间解冻 | 改为 **冻结结束楼层 = 当前楼层 + N 楼**，`当前楼层 >= 结束楼层` 解冻。 |
| 前端 `剩余冻结时间` 显示 X时Y分 | 改显示 **剩余 N 楼**。 |

## 七、疑心自然衰减

| 旧版 | 改造方案 |
|------|---------|
| 按 `上次衰减日期` 比对，过 D 天则 `疑心 -= D×2` | 改为 **每 N 楼 -X**，记 `上次衰减楼层`。 |
| 同处一室 `疑心 -1`（**已经是每回合驱动**） | 直接沿用，可作为楼层化范式样板。 |

## 八、安眠药 / 住院 → 商店道具（去时间化 + 道具化）

旧版安眠药（6h 时效 / 48h 冷却）、住院（15 天）全部是时间驱动 + AI 文本关键词触发。重置版**双重改造**：

1. **去时间化**：时效/冷却改为"剩余楼层数 / 到期楼层"。
2. **道具化**：触发方式从"AI 文本关键词"改为"商店购买 + 即时使用"（云霜凝形式）。

> 详细方案见《商店道具系统.md》。本条只作去时间化归口：道具的所有"持续/冷却"单位一律楼层。

## 十、苏文位置（已定方案：楼层驱动黑盒节律）

**已敲定：苏文位置由脚本按楼层推进推算（楼层=时间流逝），黑盒、不告知玩家/AI，不随机、不交 AI。**

- 完全楼层驱动，时间彻底退出（无例外）。
- 节律有规律但不均匀：各位置停留楼数不同；星期由楼层推算；工作日白天外出=安全期不打断；周三下午/周五上午等特例在家=刺激窗口。
- 苏文不再是"危险窗口/巡逻/疑心倒计时"的驱动源——重置版苏文是**刺激源 + 越级惩戒闸**（见《苏文系统.md》）。
- 旧版"危险时间累积→打断"逻辑**废弃**；巡逻系统**完全废除**；打断改为"低阶段硬做高阶段行为"的越级惩戒。

> 详见《苏文系统.md》§四。

---

## 十一、纯显示项（保留，但单位文案需同步改）

以下不影响计算正确性，但文案里的"小时/天/时分"需改成"楼/轮"：
- 念头进度条、难度标签
- 危险监控/打断/巡逻的剩余显示文案
- 安眠药剩余、住院剩余天数显示
- 习惯"形成时间"记录（可改记形成楼层）
- 苏文时段描述、状态参考注入（时段感可保留为氛围，不驱动逻辑）

---

## 十二、数据模型（schema）影响

需新增"楼层/轮数"字段、废弃或重定义旧时间字段，例如：
- `到期楼层` / `剩余楼数`（念头过期、道具时效、冻结、巡逻期限）
- `上次巡逻楼层` / `上次衰减楼层`
- 念头 `需要楼层数` / `开发进度(楼)`
- 危险**楼层**累计（取代危险时间累计）

> 重置版本就要重写 schema（用 Zod，对齐云霜凝），借此机会一次性楼层化，不做旧字段兼容（重置不背历史包袱）。
