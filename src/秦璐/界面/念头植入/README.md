# 念头植入界面

## 功能说明

这是一个用于植入念头的前端界面，采用**隐藏指令 + 系统消息通知**的架构，从根本上避免AI学习MVU命令格式。

## 核心设计理念

### 为什么不让AI输出MVU命令？

1. **避免AI污染**：AI一旦见过`{{SET::...}}`命令格式，可能在不合适的时候自行输出
2. **防止重复问题**：AI在回忆/总结时会重复之前的内容，容易触发二次植入
3. **符合元游戏设计**：念头植入是玩家的特殊能力，应该对AI透明

### 架构优势

```
传统方式：AI输出MVU命令 → 解析 → 执行 → 可能被AI学习和误用
新方式：  玩家操作界面 → 直接修改变量 → 日志记录 → 下次对话注入通知 → AI无法学习命令格式
```

### 日志系统：解决ROLL消息问题

**问题**：如果AI回复不满意，玩家ROLL消息重新生成时，界面刷新会导致临时的注入消息丢失。

**解决**：将念头植入操作记录到MVU变量的日志系统中（`系统追踪.念头植入日志`），而不是前端临时状态。

```typescript
// ❌ 旧方案：临时状态（ROLL后丢失）
const pendingInjections = ref<string[]>([]);

// ✅ 新方案：日志系统（持久化）
data.系统追踪.念头植入日志.push({
  目标: '秦璐',
  内容: '和儿子讨论禁忌话题',
  植入时间: '2024/11/20 18:00',
  已通知AI: false,  // ROLL后仍保留，下次生成继续注入
});
```

## 使用方法

### 在酒馆中使用

#### 1. 加载界面到消息楼层

在SillyTavern的消息楼层中插入以下HTML代码：

```html
<iframe src="dist/秦璐/界面/念头植入/index.html"></iframe>
```

或者，如果已上传到GitHub仓库，使用jsdelivr链接：

```html
<body>
  <script>
    $('body').load('https://testingcf.jsdelivr.net/gh/你的用户名/你的仓库名/dist/秦璐/界面/念头植入/index.html')
  </script>
</body>
```

#### 2. 使用界面植入念头

1. 选择目标角色（秦璐或苏梦）
2. 输入念头内容（例如："和儿子讨论禁忌话题"）
3. 点击"植入念头"按钮
4. 系统会自动计算难度并植入念头

### 实际效果

**玩家操作**：
- 在界面中输入："和儿子讨论禁忌话题"
- 点击植入

**系统处理**：
1. ✅ 直接修改变量，将念头添加到培育区
2. ✅ 自动计算难度、需要时间等参数
3. ✅ 注入系统消息：`【系统】{{user}}向秦璐的潜意识植入了念头："和儿子讨论禁忌话题"`

**AI收到的信息**：
- ✅ 看到系统消息，知道念头已植入
- ❌ 完全看不到`{{SET::...}}`命令
- ✅ 在后续剧情中通过互动开发这个念头

## 技术细节

### 工作流程

```typescript
// 1. 玩家点击"植入念头"
implantThought() {
  // 1.1 添加到念头培育区
  targetState.念头培育区.push(newThought);

  // 1.2 记录到日志（标记为"未通知AI"）
  data.系统追踪.念头植入日志.push({
    目标: '秦璐',
    内容: '和儿子讨论禁忌话题',
    已通知AI: false,  // 关键：持久化存储
  });

  saveData(); // 保存到MVU变量
}

// 2. 玩家发送消息（或ROLL后重新生成）
eventOn(CHAT_COMPLETION_PROMPT_READY, () => {
  // 2.1 从日志读取未通知的念头
  const unnotified = data.系统追踪.念头植入日志.filter(log => !log.已通知AI);

  // 2.2 生成系统消息
  const message = `【潜意识干涉】{{user}}在秦璐的潜意识深处种下了念头："${log.内容}"`;

  // 2.3 注入到prompt
  chat.splice(insertIndex, 0, { role: 'system', content: message });

  // 2.4 标记为已通知
  log.已通知AI = true;
  saveData();
});
```

### 难度自动判定

```typescript
function calculateDifficulty(content: string, targetState: any): '简单' | '中等' | '困难' {
  const stage = targetState.当前阶段;
  const moral = targetState.道德底线;

  // 低阶段 + 高道德底线 = 困难
  if (stage <= 3 && moral > 70) return '困难';

  // 高阶段 + 低道德底线 = 简单
  if (stage >= 8 && moral < 40) return '简单';

  // 其他情况 = 中等
  return '中等';
}
```

**注意**：未来可以集成完整的难度库（23种念头类型的关键词匹配）

### 时间计算

- 简单：3小时
- 中等：4小时
- 困难：5小时

过期时间自动设置为植入时间+24小时

### 三重保障

1. **Schema层去重**：`schema.ts`中的transform自动过滤内容相同的念头
2. **自然时间隔离**：玩家手动操作不会像AI那样频繁重复
3. **日志系统**：即使ROLL消息，念头植入通知也不会丢失

## 与世界书配合

世界书已更新为强调**玩家操作**而非**AI命令**：

```yaml
植入操作说明: |-
  【重要】念头植入是玩家的元游戏操作，由玩家通过界面完成。
  AI不需要也不应该输出任何植入命令或相关的MVU指令。

  【AI的职责】：
  1. 当看到【系统】消息提示"{{user}}向XX植入了念头：YYY"时
  2. 了解这个念头已经被植入到对应角色的潜意识中
  3. 在后续剧情中通过相关互动来"开发"这个念头
  4. 开发进度会自动累积，AI只需专注于剧情演绎
```
