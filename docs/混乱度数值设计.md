# 混乱度数值设计

> 创建日期：2026-01-17
> 状态：已设计

## 概述

记忆混乱度是一个威胁数值，用于追踪赵霞精神状态的稳定性。当混乱度达到100时，触发精神崩溃结局。

**核心设计理念**：
- 混乱度在梦境场景中**逐步积累**，不是从0开始
- 玩家在进入场景5时，混乱度已经处于较高水平（约80）
- 场景5内根据**记忆连贯性**使用不同的增量计算

---

## 混乱度演变

### 梦境场景入口设置

**重要**：混乱度只在真相模式（进入过梦境）的状态栏显示，纯爱模式下不显示。

| 进入场景 | 混乱度设置 | 说明 |
|---------|-----------|------|
| 进入场景1 | 40 | 首次进入梦境，记忆开始被触动 |
| 进入场景2 | 60 | 记忆深入，混乱累积 |
| 进入场景3 | 80 | 接近临界点 |
| 进入场景4 | 不变 | 场景4时间线在场景5之后，不影响 |
| 进入场景5 | 80（固定） | 无论之前走什么路线，统一为80 |

**注意**：
- 混乱度在进入梦境时设置，不是退出时
- 场景5固定为80，与玩家之前的路线无关
- 这确保所有玩家在场景5有相同的起点
- initvar.yaml 中的初始值0只是占位符，实际值在进入场景时设置

### 为什么场景5固定为80？

1. **公平性**：无论玩家走什么路线，场景5的挑战难度一致
2. **容错设计**：留出20点空间给2-3次越轨机会
3. **避免复杂计算**：不需要追踪玩家的具体路线

## 场景5内的混乱度增加

### 越轨行为

在场景5中，以下行为会增加混乱度：

| 行为类型 | 路线 | 增量 | 说明 |
|---------|------|------|------|
| 性行为 | 完美路线（连贯性=3） | +7 | 3次到达100 |
| 性行为 | 非完美路线（连贯性<3） | +10 | 2次到达100 |
| 打断仪式 | 所有路线 | =100 | 直接设为100 |

### 计算示例

**完美路线玩家**（记忆连贯性=3）：
```
初始: 80
第1次越轨: 80 + 7 = 87 (警告)
第2次越轨: 87 + 7 = 94 (警告)
第3次越轨: 94 + 7 = 101 ≥ 100 → 触发精神崩溃
```

**非完美路线玩家**（记忆连贯性<3）：
```
初始: 80
第1次越轨: 80 + 10 = 90 (警告)
第2次越轨: 90 + 10 = 100 → 触发精神崩溃
```

**打断仪式**：
```
任意混乱度 → 直接设为100 → 触发精神崩溃
```

## 与现有系统的关系

### 替代次数计数

原有 `confusionEndingSystem.ts` 使用 `混乱结局.性行为次数` 来追踪：
- 未完成场景3：2次触发
- 已完成场景3：3次触发

**新设计**：改用混乱度数值表示同样的机制：
- 非完美路线：80 + 10×2 = 100（2次）
- 完美路线：80 + 7×3 = 101（3次）

### 状态栏显示

混乱度会在真相模式的状态栏中显示，玩家可以直观看到：
- 当前危险程度
- 距离100还有多少空间

## 技术实现要点

### 1. 进入梦境时设置混乱度

在 `dreamKeywordDetection.ts` 或相关入口函数中：

```typescript
function setConfusionOnDreamEntry(data: SchemaType, sceneNumber: number): void {
  const confusionValues: Record<number, number> = {
    1: 40,
    2: 60,
    3: 80,
    4: data.梦境数据.记忆混乱度, // 不变
    5: 80, // 固定值
  };

  const targetValue = confusionValues[sceneNumber];
  if (targetValue !== undefined && data.梦境数据.记忆混乱度 < targetValue) {
    data.梦境数据.记忆混乱度 = targetValue;
    console.info(`[混乱度] 进入场景${sceneNumber}，混乱度设为${targetValue}`);
  }
}
```

### 2. 场景5越轨增量计算

在 `confusionEndingSystem.ts` 中：

```typescript
function getViolationIncrement(data: SchemaType): number {
  const coherence = data.梦境数据.记忆连贯性 ?? 0;
  return coherence === 3 ? 7 : 10;
}
```

### 3. 打断仪式处理

```typescript
function handleWeddingInterruption(data: SchemaType): void {
  data.梦境数据.记忆混乱度 = 100;
  console.info(`[混乱度] 打断仪式，混乱度直接设为100`);
}
```

## 与其他混乱度增加机制的关系

根据 KNOWN_ISSUES.md 的设计，以下情况**不会**在梦境中增加混乱度：
- 危险内容检测（梦境中不生效）
- 境界打断系统（梦境中不生效）
- 部位开发（场景1-4允许）

唯一在梦境中增加混乱度的情况：
- 场景1-4错误重构：+20（但被完美记忆路线豁免）
- 场景5越轨行为：+7 或 +10
- 场景5打断仪式：=100
