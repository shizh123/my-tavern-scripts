## v0.06 — 修正 AI/脚本职责边界 + 恢复状态栏美化 + 补 options 条目

**背景**：v0.05 让 AI 写依存度 delta 和货币"+100"提示是错的——依存度应由脚本在念头成熟时结算，货币结算是脚本内部事情不该告诉 AI。同时状态栏美化版（`th-stage`/`bg-glow`/`st-mark`/脉冲特效）在 worktree reset 中丢失。

**改动**：
- 依存度从 AI 白名单移除，改为脚本在念头成熟时结算
- 货币不再暴露"+100"细节给 AI，只标脚本管理
- 从 Kilo session diff 反向 patch 恢复美化版 App.vue 的 template + style，保留 v0.05 script
- 补 `<options>` 世界书条目（`系统设定/行动选项.yaml`）
- `initvar`/`schema` 恢复默认心理想法（秦璐/苏梦各一句，用「儿子」而非固定名）
- 设计文档同步更新

## v0.09 — 对齐云霜凝：注册 Schema + defineMvuDataStore

**背景**：v0.08 仍无法把状态快照注入 prompt，旧楼层还显示新状态。

**根因**：
1. 脚本没等 Mvu 就绪、没 `registerMvuSchema(Schema)`
2. 前端硬编码 `message_id: -1`，所有楼层 iframe 都读最新楼

**改动**：
- 脚本入口改 `async IIFE` + `waitGlobalInitialized('Mvu')` + `registerMvuSchema(Schema)`
- 加 `dryRun` 守卫（后面又出问题，见 v0.11/0.13）
- 快照插入策略对齐云霜凝：末尾为 assistant prefill 则插到之前，否则 push 到末尾
- 新增 `store.ts` 使用 `defineMvuDataStore(Schema, () => ({ message_id: getCurrentMessageId() }))`
- App.vue 从 `Mvu.getMvuData` 硬编码切到 store.data

## v0.11 — 移除 dryRun 守卫（错误决策，v0.13 恢复）

**当时判断**：用户报告 `dryRun=true，跳过注入`，误以为该酒馆版本 dryRun 永远为 true。

**改动**：去掉 `if (dryRun) return` 允许 dryRun=true 时也注入。

**后果**：见 v0.13——实际是有两次调用，dryRun=true 那次应该跳过。

## v0.12 — 修 message_id 越界

**背景**：v0.11 之后报错 `提供的消息楼层号 '5' 超出了范围 [-5, 5)`。

**根因**：`getCurrentFloor()` 返回 `chat.length = 5`，但 Mvu 的 `message_id` 是数组下标 `[-5, 5)`，5 越界。

**改动**：`Mvu.getMvuData` 用 `message_id: -1`（最新楼）；`getCurrentFloor()` 仍返回 `chat.length` 用于楼层判定逻辑。

---

## v0.13 — 修 isInVulnerableWindow 导入错误 + 恢复 dryRun 守卫

**背景**：v0.12 之后报错 `isInVulnerableWindow is not a function`。

**根因 1**：`isInVulnerableWindow` 定义在 `thoughtEngine.ts`，但 `index.ts` 从 `suwenRoutine` 导入，运行时为 undefined。

**根因 2**：v0.11 误移除 dryRun 守卫。实际酒馆每次生成会调用 CHAT_COMPLETION_PROMPT_READY 两次，第一次 dryRun=true 是预热应跳过，第二次 dryRun=false 才是真实生成。

**改动**：
- 修正 `isInVulnerableWindow` 从 `thoughtEngine` 导入
- 恢复 dryRun 守卫，与云霜凝一致

## v0.15 — 植入上限 + 快照瘦身 + 在场角色机制 + 界面重设计

### 1. 每楼植入上限 3 条

**背景**：植入无频率限制，一轮内可无限灌念头，会淹没 AI 判定通道。

**改动**：
- `thoughtEngine` 新增 `MAX_IMPLANTS_PER_FLOOR = 3` + `countImplantsAtFloor()`（按 `植入楼层 === 当前楼层` 计数，秦璐+苏梦合计）
- `implantThought()` 超限拒绝（返回 null）；状态栏植入前拦截 + 额度点显示
- 规则：AI 回复后楼层推进额度自动刷新；ROLL 不推进楼层、不刷新额度

### 2. 状态快照瘦身（去三重重复）

**背景**：用户实测发现快照大量冗余：baseline 样板每条念头重复一遍、判定任务 B 把念头再列一遍、判定规则与常驻世界书整段重复、玩家侧信息（货币/习惯栏/待腾位）混入。

**改动**（原则：只注入脚本管理且 AI 无法从上下文推出的信息）：
- 删：堕落度数字/依存度/情绪/位置/内心/货币/习惯栏容量/已成熟待腾位/⚡越级植入提示
- 保留并瘦身：阶段行（+心防松动⚡后缀，脚本覆写 AI 上下文里没有）、苏文位置（脚本黑盒作息，快照是唯一通道）
- 念头影响块：语义样板交给世界书「念头习惯表现」（已手动转蓝灯常驻，index.yaml 同步 `constant: true`），快照只列清单，念头附 ID
- 判定任务 A/B：只列条目+一行路径提示，完整规则依赖常驻「变量输出格式」；任务 B 不再重复列念头（引用影响块 ID）

### 3. 在场角色机制（对标云霜凝 _当前场景角色）

**背景**：`系统.当前角色` 初始化后无人写入（UI 只改本地状态、AI 无权、脚本不写），快照永远只注入秦璐——苏梦的念头永远进不了判定任务 A，会错过 3 楼窗口卡死在判定中。

**改动**：
- schema 加 `系统.在场角色 {秦璐:bool, 苏梦:bool}`，`当前角色` 标记 @deprecated
- AI 每轮维护（进场/离场/复核），世界书「变量更新规则」「变量输出格式」补白名单和维护规则；initvar 补初始值
- 快照按在场过滤：状态行/影响块/判定任务 B 只给在场角色，同时在场则合并；**判定任务 A 不过滤**（类型判定纯语义，过滤会让提前埋的念头错过 3 楼窗口），改为两角色都列、路径逐条附
- 心防松动覆写改为对所有在场角色生效；脚本兜底：全 false 按秦璐在场

### 4. 状态栏界面重设计

**背景**：旧版满屏发光无层次、三层动画背景、✦ 装饰陈旧、桌面优先两栏在移动端总是塌单列。

**改动**（App.vue template+style 重写，script 逻辑不变）：
- 保留阶段驱动主题色机制（5 阶段暗金→电紫），变量精简为 acc/acc2/bg/panel/line
- 移动端优先单列流：顶栏(世界+货币) → 角色 tab(含在场绿点) → 阶段/情绪/三维 hero → 内心独白(衬线引文) → 植入(含 3 点额度计量) → 念头/习惯/苏文/仪容卡片；≥640px 双列
- 光效只留心防松动（唯一高调元素）；UI 数据无衬线，念头/习惯内容衬线"她的笔迹"

## v0.17 — 培育槽 + 阶段禁区 + 界面 tab 分页重构（frost 皮肤）

**背景**：①每楼3条的流量限制过松，连续几楼植入可积压大量培育中念头；
②低阶段角色被念头影响到几乎改性格——影响本身不设门槛（这是核心爽点），
但行为/话题的出口需要阶段限幅；③用户不满上版极简样式，要求层次照云霜凝
（毛玻璃/渐变/底纹），商店独立成页。

**改动**：
- **培育槽制取代每楼限制**：判定中+培育中 计槽，每角色各 3 槽（植入扩容特权 → 4）；
  删除 `MAX_IMPLANTS_PER_FLOOR/countImplantsAtFloor`，新增 `countActiveThoughts/getCultivationSlots`；
  退回放宽：判定中/培育中/未达标/已过期 均可退回腾槽（已成熟不可）
- **阶段禁区**：`STAGE_RESTRAINTS` 5 档"允许+禁止"清单，按在场角色动态注入影响块末尾；
  原则：念头决定她"想什么"，阶段决定她"敢什么"——影响恒在，出口限幅；
  世界书「念头习惯表现」重写（删"脱口聊到"，接入阶段约束语义，**需手动同步酒馆**）
- **界面重构**（对标云霜凝架构，代码省一半）：
  - App.vue = frost 三明治（多段渐变底+斜纹底纹+径向光晕+毛玻璃顶栏）+ tab 分页
    （秦璐/苏梦/苏文/网店），阶段换色暖调保留（主题随最近选中的角色 tab）
  - CharPanel 单组件双角色共用（云霜凝是 YunPanel/LuoPanel 两份重复代码）
  - SuwenPanel 独立页（双疑心条 + 冻结徽章）；ShopPanel 独立页（"为谁选购"切换 +
    类别分页 + 固定详情栏 + 秦/梦拥有者徽章）
  - 老楼层内容区默认折叠（对标云霜凝 v2.0.31 性能优化）

## v0.22 — 0.21 试用反馈五连修（已发布）

**反馈与根源**：
1. AI 对换装困惑（历史正文=毛衣棉裤 vs 变量=连衣裙）——三重根源：换装只改变量没有叙事桥；
   `变量输出格式` 白名单没列 服装细节/妆容细节（AI 判断"不得修改"是对的）；穿着信息注入了三处互相冗余
2. UI 仪容卡"装备中"标签与下方服装字段重复
3. 装备可购买但阶段不足无法装备（buyEquipment 没吃阶段门槛）
4. 服装道具正文零表现——同 1 根源，AI 收到快照但选择跟随历史正文
5. 苏文疑心值从未上涨——**非 bug**：疑心累积规则本来就未实现（待设计），与测试暂停项无关

**改动**：
- 换装叙事桥：每次装备/卸下注入一次性换装事件（首穿用专属文案，复穿/卸下用通用文案），
  `queueItemEvent` 统一事件队列
- 穿着单一事实源 = 服装/妆容细节变量：删快照【穿着】行、删 UI"装备中"标签行；
  妆容细节新增 `香氛` 字段，雾致香水/催情香水补进 CLOTHING_WRITE（22 件全映射）；
  仪容卡增列 配饰/装饰/唇妆/眼妆/妆饰/香氛（"无"不占行）
- 世界书：`变量更新规则` 增「换装机制」条目 + 服装/妆容细节入 AI 可更新字段；
  `变量输出格式` 白名单补 服装细节/妆容细节（**两条都需手动同步酒馆**）
- 能买即能穿：buyEquipment + ShopPanel 购买按钮同吃阶段门槛（未达显示"需阶段N"）

**装备接入主循环（"道具鸡肋"讨论产物）**——被否决的方向：疑心挂苏文作息时机
（逼玩家跳换装舞、烧楼层篇幅、违背 AIRP 自由度）；采纳 C+D：
- C 装扮意识动态提示：脚本检测在场角色装备中**风险≥1**的物品（正常衣物如修身连衣裙/
  红绳手链不触发），快照注入一行表现指令（异物感/遮掩/走神），满星追加"从内到外都是
  他置办的"意识；哪件该提示靠调整该物品风险值控制
- D 仪容星标·全套加成：5 星 = 4 装备槽各有网店装备 + 任意体改（`getOutfitStars`）。
  满星 = 所有培育中念头 +1/楼（无视类型匹配）**+ 苏文对她疑心 +1/楼**（高收益高风险，
  用户提议的代价耦合）；不满星疑心每楼回落 0.5；借口短信冻结期不涨不落、到期自动解冻
  （消耗品从此有实际用途）。疑心值纳入回滚保护（AI 不可改，脚本结算）。
  UI：仪容卡 5 星行，满星点亮 + 脉冲动画 + "全套共鸣"注记。数值全部待平衡期统一调。

**疑心系统完善（阶梯 + 坏结局）**：
- 阶梯提示（快照动态注入，AI 只演态度不改数值）：30 起疑（多看两眼/状似无意盘问）、
  60 警觉（盘问变多/留意行踪手机/气氛发紧）、90 临界（阴沉冷淡/暗中翻查/一触即发）；<30 不注入
- 触顶 100 → `系统._坏结局 = '疑心爆表·角色名'` 锁定：快照只注入终局指引块（培育/判定停止，
  演绎摊牌后果）；引擎写阶段全停（回滚保护仍生效）；植入/商店/出售全部拒绝（"结局已锁定"）；
  状态栏顶部红色脉冲横幅"☠ 坏结局已锁定"
- 满星装扮意识句改**动态生成**（列实际装备+体改清单），并追加知情隔离标注：
  "此信息仅属于她的私密认知，苏文并不知情（除非正文已揭穿）"——不做来历/路线的叙事断言，
  防止写苏文心理的 AI 把信息泄进苏文视角（用户指正 ×2）
- 疑心值可能出现 0.5 步进，速览条/苏文卡显示取整

**培育速度平衡（原待办，本版定稿）**：
- 基础需要楼数 3/6 → **简单 5 / 困难 10**
- 新增**满配下限** `MIN_MATURE_FLOORS`：距植入不足 简单3/困难4 楼不成熟（进度先攒着，
  楼数到了才结算）——满星+装备+相关度全堆也无法瞬发，"冲刺"保留过程感
- 满配节奏：简单 3 楼 / 困难 4 楼；纯放置：5 / 10 楼
- **调试满星后门**：状态栏星标区 1.5s 内连点 5 次切换 `系统._调试满星`（MVU 持久，防 reload），
  开启时两角色均按满星结算（星全亮 + "⚙ 调试满星"注记；培育+1/楼、疑心+1/楼照常）——
  正式数值落地后测试快节奏专用，发正式版前记得确认关闭状态

## v0.37 — 玩家反馈批次：打断重复触发修复 + 越级封顶 + 手动晋阶（tag 0.37）

玩家反馈五条，本版落三条；"常识改变玩法"（认知歪曲轴）用户定为下版立项；
"五阶段太快"先靠手动晋阶观察体感再决定动阈值。

**1. 修 BUG：疑心10·打断同档重复触发**

根因：档位标记/冷却是 MVU 楼级字段没错，但重roll时 MVU 从上一楼重建变量，
全靠 v0.28 的**模块内存**保护快照并集回来——页面刷新/切聊天/重启后快照归零，
此时重roll打断楼 → 标记+12楼冷却全丢 → 同档重发（模块变量幂等门控陷阱，
这次踩坑的是保护快照自己）。

修法：`syncInterruptMirror()`——打断状态（档位/冷却/余波/苏文视角+楼层戳）
镜像到 **chat 级变量** `秦璐重置版_打断镜像`，每次捕获快照时：
镜像楼层 ≤ 当前楼 → 并集回填内存快照（rollback 沿用既有并集/取大逻辑）；
镜像楼层 > 当前楼 = 玩家主动回档到打断前 → 不回填、直接覆盖（保留回档重演语义）。
苏文视角按"活跃优先"合并（待看或剩余楼>0 才算活跃，防复活已看完的 POV）。

**2. 越级加成封顶 +2（阶段闸门被玩家指"形同虚设"）**

此前 心防松动+1 / 裂纹+1 / 头孢酒+2 / 越级钥匙+1 可叠到 +5——阶1能收阶5念头。
getEffectiveStage 重构：越级钥匙并入函数内（原在两处调用点外挂），
加成总和 `min(2, Σ)` 封顶。UI 同步：未达标退回显示具体原因
（「类型」通常要到第N阶段+当前阶段，title 注明封顶规则），CATEGORY_STAGE 导出。

**3. 手动晋阶（替代自动跃迁，玩家提议）**

- 堕落度越过下一阶段门槛后**不再自动跳阶**：五处自动跳阶全拆
  （念头成熟/体改/观影/习惯腾位补转 → 只记待晋日志；每楼阶段校正 → 只向下钳制防超前）
- 状态栏阶段区亮金色呼吸按钮「晋阶 ▲」（坏结局不亮；旧楼只读全局已挡）→
  `promoteStage()`：一次一阶（跨两档点两次、各演一次转变），
  注入方向性晋阶事件（"她终于承认了已经发生的变化"，不写死动作）
- 滞留低阶自带代价：高阶念头被闸门退回、高阶商品买不了——不构成刷子策略
- 旧档零迁移：当前阶段==派生档的档不受影响；此后堕落度先涨、阶段等玩家点

无世界书改动。数值不动（阈值 20/40/60/80 保持，观察手动晋阶后的节奏体感）。

**v0.37 补：手动"重新处理变量"改为只恢复不推进（玩家实测漂移）**

此前 VARIABLE_UPDATE_ENDED 守卫在非 AI 生成周期直接跳过 → 手动重处理时 MVU 按
AI 指令从上一楼重建，苏文位置/疑心/货币等脚本管理字段全部漂移（玩家看到
"生成显示在家、重处理变外面，其他变量也变"——重处理后的值不是作息真值）。
改为：非生成周期若有内存快照 → `rollbackProtectedFields` 拉回真值写回（幂等，
可反复点），引擎不推进；刷新后无快照则维持原状。
另：玩家澄清"苏文位置对不上"是旧版本未更新（v0.29 已修的滞后），时段跳转词
扩充（清晨/傍晚/晚上等，现仅认"第二天"系）降级为可选优化，攒着后做。
