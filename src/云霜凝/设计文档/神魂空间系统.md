# 神魂空间系统设计文档

## 概述

神魂空间是 {{user}} 圣阳之体与云霜凝极寒体质产生共鸣的精神领域。
1-2阶段治疗强制在神魂空间进行，3阶段起玩家可自由选择神魂空间或现实互动。

---

## 实现方案

### 技术架构（参考赵霞梦境，但简化）

| 对比项 | 赵霞梦境 | 云霜凝神魂空间 |
|--------|---------|-------------|
| 入口触发 | 关键词检测（"进入梦境"等） | UI 按钮主动点击 |
| 场景数量 | 5个不同记忆场景 | 单一空间，随阶段变化 |
| 退出机制 | 时间（10:00自动退出） | 手动切换 or 治疗结束 |
| 复杂度 | 高（记忆连续性、摘要生成） | 中（环境描写 + 阶段变化） |

## 1. 状态栏入口按钮（App.vue）

位置：治疗进度条下方、标签页之前。

```
┌─────────────────────────────────┐
│ 第X天 · 08:00 · 灵石 XX石        │ ← 顶栏
│ 阶段1·破冰  [████░░] 23.5%       │ ← 治疗进度
├─────────────────────────────────┤
│ [⟨ 进入神魂空间 ⟩] [○ 现实互动]  │ ← 模式入口栏
├─────────────────────────────────┤
│ 云霜凝 | 苗广 | 商店              │ ← 标签页
```

**阶段逻辑：**
- 阶段 1-2：只显示"进入神魂空间"按钮（强制，不显示"现实互动"）
- 阶段 3+：两个按钮都显示，可自由切换

**点击效果：**
- 设置 `store.data._当前互动模式 = '神魂空间'`（或 '现实互动'）
- 设置 `store.data._神魂空间激活中 = true/false`
- 设置 `store.data._待发送道具事件 = '__神魂空间入口__'`（触发入口叙事）

```typescript
'__神魂空间入口__': (d) => d.云霜凝.心理防线 > 70

```

## 3. 场景指引注入（promptInjection.ts → buildStatusSnapshot）

新增函数 `getSoulSpaceGuidance(data, stage)`，在神魂空间模式下附加到状态快照：

```typescript
function getSoulSpaceGuidance(data: SchemaType): string {
  if (data._当前互动模式 !== '神魂空间') return '';
  const stage = data.治疗.阶段;

  return `
【神魂空间场景指引】
圣阳之体与极寒体质的共鸣领域。环境呈淡紫色与冰蓝色光晕，霜晶纹路在空间中
流动，代表寒毒的具象化。云霜凝的神魂以半透明形态显现。

治疗机制：
- 圣阳温热以光晕形式渗入霜晶，体现为"霜纹松动/融化"的意象
- 心理防线在此更薄弱（意识直接暴露），但仍有自我意志
${stage <= 2 ? '- 阶段1-2：这是唯一治疗场所，云霜凝无法完全拒绝进入' : ''}
${stage >= 3 ? '- 阶段3+：云霜凝已有一定熟悉感，不再像初期那样强烈抵触' : ''}

描写重点：
- 触感是精神层面的，比现实更直接传达情绪
- 寒毒对抗体现为"冰晶←→暖光"的视觉意象
- 空间随云霜凝的情绪变化（防线高→结霜密集，信任高→光晕扩散）
`;
}
```

调用位置：在 `buildStatusSnapshot` 末尾追加 `getSoulSpaceGuidance(data)` 的返回值。
