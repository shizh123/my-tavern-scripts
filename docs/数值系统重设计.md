# 数值系统重设计

> 创建日期：2026-01-16
> 状态：设计阶段

## 设计理念

### 核心原则

**数值是"假的"，但玩家要觉得是自己"玩"出来的**

- 游戏是半线性的，只有5天游玩时间
- 数值的作用是给玩家进度反馈，不是真正的模拟系统
- 确保刚好在 Day 4-5 能完成预期数值，触发结局变化

### 之前遇到的问题

| 问题 | 描述 |
|------|------|
| 进度太快 | Day 3 就已经开发完了 |
| 数值不影响AI | 数值变了但AI描写没变化 |
| 计算逻辑混乱 | 数值莫名其妙地跳动 |

## 部位进度系统

### 部位分类

| 部位 | 可开发场景 | 说明 |
|------|-----------|------|
| 嘴巴 | 场景1-4 | 身体部位 |
| 胸部 | 场景1-4 | 身体部位 |
| 下体 | 场景1-4 | 身体部位 |
| 后穴 | 场景1-4 | 身体部位 |
| 精神 | **仅场景5** | 精神部位，场景1-4无法开发 |

**重要约束**：
- 场景1-4 只能开发身体部位（嘴巴/胸部/下体/后穴）
- 场景5 只能开发精神，无法开发身体部位
- 不同部位的数值不能同时增长相同数值（避免看起来像假数据）

### 玩家选择部位的意义

- 进入梦境前玩家选择部位
- 选择影响最终结局的变化
- 需要保留这个机制

## 触发机制设计（方案D修正版）

### 触发条件

进度增长需要满足以下**任意一个**条件：

```
1. 玩家输入包含宽泛关键词（主要触发方式）
2. AI回复包含 [PROGRESS] 标记（需要二次验证）
```

### 关键词触发（条件1）

玩家输入包含相关关键词时触发：

| 部位 | 关键词示例（宽泛） |
|------|-------------------|
| 嘴巴 | 亲、吻、舔、嘴、唇、口、含... |
| 胸部 | 胸、乳、揉、捏、摸胸... |
| 下体 | 下面、私处、摸、插、进入... |
| 后穴 | 后面、屁股、后穴、肛... |
| 精神 | （场景5专用，由步骤系统控制） |

**注意**：关键词列表需要后续细化，保持宽泛但不过于宽泛。

### AI报告触发（条件2）

AI在回复末尾输出标记：
```
[BODY_PROGRESS: 嘴巴+1, 胸部+1]
```

**二次验证规则**：
```
AI报告 [PROGRESS: 某部位]
→ 检查玩家输入是否有"互动意图"
→ 如果玩家输入完全无关（如"看风景"），忽略AI报告
→ 如果玩家输入有互动意图（如"靠近她"），接受AI报告
```

**互动意图关键词**（宽泛）：
- 靠近、接近、抱、拉、牵、碰、触...
- 任何暗示物理接触或亲密行为的词

### 不触发的情况

| 玩家输入 | AI报告 | 结果 |
|---------|--------|------|
| 无关键词（"看风景"） | 无 | +0% |
| 无关键词（"看风景"） | 有 | +0%（验证失败） |
| 有关键词（"亲吻"） | 无 | +5%（关键词触发） |
| 有关键词（"亲吻"） | 有 | +5%（不叠加） |
| 有互动意图（"靠近她"） | 有 | +5%（AI报告通过验证） |

## 增长数值控制（脚本控制）

### 单次增长限制

| 限制类型 | 数值 | 说明 |
|---------|------|------|
| 单部位单轮上限 | 5% | 无论触发几次，每轮每部位最多+5% |
| 同一部位不叠加 | - | 关键词+AI报告同时触发，只算一次 |
| 不同部位可同时增长 | - | 但增量要有差异（如嘴巴+5%，胸部+3%） |

### 每晚上限

| 部位类型 | 每晚上限 | 说明 |
|---------|---------|------|
| 身体部位（嘴巴/胸部/下体/后穴） | 20-25% | 防止单晚速通 |
| 精神（场景5） | 由步骤系统控制 | 见场景5专属规则 |

### 预期游戏节奏

| 天数 | 预期进度 | 说明 |
|------|---------|------|
| Day 1 | 0-25% | 首次进入梦境 |
| Day 2 | 25-50% | 继续开发 |
| Day 3 | 50-75% | 主要开发期 |
| Day 4 | 75-100% | 完成开发 |
| Day 5 | 结局判定 | 根据完成度决定 |

## 场景5 精神进度（特殊规则）

场景5使用12步线性剧情系统，精神进度由步骤完成决定：

| 事件 | 精神增量 |
|------|---------|
| 首次进入 | +15% |
| 每完成一步（12步） | +5%（共60%） |
| 契合度高的步骤 | 额外+2% |
| 第二次进入 | +10% |

**最大精神进度**：15 + 60 + 24（契合度奖励）+ 10 = ~100%

**注意**：场景5的精神进度不受关键词系统控制，完全由步骤系统管理。

## 数值显示与AI影响

### 显示给玩家

状态栏显示具体的部位进度百分比：
- 嘴巴：45%
- 胸部：30%
- 下体：25%
- 后穴：10%
- 精神：60%

### AI提示词（简化）

不给AI具体数值，而是给**等级描述**：

| 进度范围 | 等级 | AI提示 |
|---------|------|--------|
| 0-19% | Lv0 未开发 | 完全抗拒，会明确拒绝 |
| 20-39% | Lv1 初步觉醒 | 轻微敏感，有些抗拒但不会强烈拒绝 |
| 40-59% | Lv2 明显敏感 | 开始接受，抗拒中带着期待 |
| 60-79% | Lv3 高度敏感 | 主动配合，轻微刺激就有强烈反应 |
| 80-100% | Lv4 完全开发 | 完全沦陷，会主动寻求 |

这样AI只需要根据等级调整描写风格，不需要理解具体数值。

## 威胁数值

### 记忆混乱度（已确定）

#### 设计理念

混乱结局 = 玩家"明知故犯"的惩罚
- 玩家**知道**这是结婚日记忆，应该按剧情走
- 玩家**故意**触发不合适的行为（性行为等）
- 玩家**无视**AI警告继续做
- → 触发混乱结局（坏结局，游戏需要重置）

#### 场景5 混乱结局触发规则

**场景时间线回顾**：
- 场景3（23岁）：玩家与赵霞发生初夜
- 场景5（25岁）：赵霞与苏文结婚日

**核心逻辑**：已完成场景3 = 赵霞记忆中有与玩家的性关系 = 更多容错

| 前置条件 | 性行为次数限制 | 说明 |
|---------|---------------|------|
| 未完成场景3 | 2次触发混乱 | 第1次警告，第2次混乱结局 |
| 已完成场景3 | 3次触发混乱 | 有记忆基础，多1次容错 |
| 打断婚礼仪式 | 直接触发 | 无论任何条件，直接混乱结局 |

#### 性行为检测关键词

性行为 = 任何涉及性器官的性行为（下体性行为、口交、乳交等）

**关键词分类**：

| 类型 | 直接关键词（单独出现即触发） |
|------|---------------------------|
| 下体性行为 | 插、进入、操、干、肏、交合、贯穿、抽插... |
| 口交 | 口交、深喉、吞精... |
| 乳交 | 乳交、胸推... |

| 类型 | 组合关键词（动作+部位才触发） |
|------|---------------------------|
| 动作词 | 舔、吸、含、夹、吞、套弄... |
| 部位词 | 下面、私处、阴、肉棒、鸡、龟头、阳具、性器... |

**检测逻辑**：
```typescript
const 性行为关键词 = {
  // 直接触发（单独出现就算）
  直接: ['插', '进入', '操', '干', '肏', '交合', '贯穿', '抽插',
         '口交', '深喉', '吞精', '乳交', '胸推'],

  // 组合触发（需要配合部位词）
  组合: {
    动作: ['舔', '吸', '含', '夹', '吞', '套弄'],
    部位: ['下面', '私处', '阴', '肉棒', '鸡', '龟头', '阳具', '性器']
  }
};

function 检测性行为(输入: string): boolean {
  // 1. 直接关键词检测
  if (性行为关键词.直接.some(k => 输入.includes(k))) return true;

  // 2. 组合检测：动作 + 部位
  const 有动作 = 性行为关键词.组合.动作.some(k => 输入.includes(k));
  const 有部位 = 性行为关键词.组合.部位.some(k => 输入.includes(k));
  return 有动作 && 有部位;
}
```

**与部位开发关键词的关系**：
- 部位开发关键词（宽泛）：用于场景1-4的部位进度增长
- 性行为关键词（严格）：仅用于场景5混乱检测
- 两者有重叠但用途不同

#### 婚礼仪式打断检测

**打断关键词**（在步骤3-5期间检测）：
- 拉走、带走、阻止、抢走、打断、中止、停下...
- 配合场景5步骤系统：步骤3-5是仪式阶段

**检测逻辑**：
```
if (当前步骤 in [3,4,5] && 玩家输入包含打断关键词) {
  → 直接触发混乱结局
}
```

#### 触发流程（更新版）

**未完成场景3**：
```
第1次性行为 → +20% + AI警告
"赵霞的眼神变得迷茫，这段记忆似乎被什么东西干扰了..."

第2次性行为 → 直接混乱结局
"赵霞的意识彻底崩溃，记忆碎片四散..."
```

**已完成场景3**：
```
第1次性行为 → +10% + 轻微警告
"赵霞有些困惑，但记忆中似乎有类似的片段..."

第2次性行为 → +15% + AI警告
"赵霞的记忆开始扭曲，她分不清这是回忆还是幻觉..."

第3次性行为 → 直接混乱结局
"赵霞的意识彻底崩溃..."
```

**打断婚礼仪式**：
```
步骤3-5期间打断 → 直接混乱结局
"婚礼被强行中断，赵霞的记忆出现严重错乱..."
```

#### 玩家路线对比（更新版）

| 玩家类型 | 路线 | 结果 |
|---------|------|------|
| 正常玩家 | 按12步剧情走 | ✅ 安全 |
| 完美玩家 | 完成1-2-3 + 场景5正常 | ✅ 很安全 + 最佳结局 |
| 性急玩家（无场景3） | 场景5尝试2次性行为 | ❌ 混乱结局 |
| 性急玩家（有场景3） | 场景5尝试3次性行为 | ❌ 混乱结局 |
| 破坏玩家 | 打断婚礼仪式 | ❌ 直接混乱结局 |

#### 关键点

- 正常玩家：**不会触发**（按剧情走就没事）
- 完成场景3：**多1次容错**（有记忆基础）
- 打断仪式：**无容错**（直接结局）
- 关键词共享：性行为检测复用下体部位关键词库
