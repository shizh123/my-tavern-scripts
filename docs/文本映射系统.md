# 文本映射系统

> 创建日期：2026-01-17
> 文件位置：`src/赵霞/脚本/游戏逻辑/dualTrackSystem.ts`
> 状态：已实现

## 概述

文本映射系统实现了"纯爱模式"与"真相模式"之间的界面文本动态切换。基于`已进入过梦境`变量，同一个数值在不同模式下会显示不同的名称，实现玩家认知落差效果。

---

## 设计理念

游戏逻辑完全不变，只是显示的文字不同：
- **纯爱模式**（未进入梦境）：伪装成纯爱恋爱游戏
- **真相模式**（进入梦境后）：显示真实的NTR术语

## 核心函数

### `getDisplayText(key, data)`

获取指定键的显示文本。

```typescript
export function getDisplayText(key: keyof typeof 文本映射表, data: SchemaType): string | null {
  const 当前模式 = data.世界.已进入过梦境 ? '真相' : '纯爱';
  return 文本映射表[key][当前模式];
}
```

### `getRealmName(境界等级, data)`

获取境界显示名称。

```typescript
export function getRealmName(境界等级: number, data: SchemaType): string {
  const key = `境界${境界等级}` as keyof typeof 文本映射表;
  return getDisplayText(key, data) ?? `境界${境界等级}`;
}
```

### `getBodyPartName(部位名称, data)`

获取部位显示名称。

### `shouldShowThreatValues(data)`

检查是否应该显示威胁数值（只在真相模式显示）。

> **注意**：苦主视角显示条件由 `dualTrackSystem.ts` 的 `shouldGenerateHusbandThought()` 控制，条件为：真相模式 + 境界≥2 + 日常阶段 + 苏文在家。

## 系统提示映射

根据模式生成不同的系统提示文本：

```typescript
export const 系统提示映射 = {
  依存度提升: {
    纯爱: (value) => `赵霞对你的好感提升了！(+${value})`,
    真相: (value) => `赵霞对你的依存度提升，她越来越需要你。(+${value})`,
  },
  部位开发: {
    纯爱: () => `你们的关系更加亲密了。`,
    真相: (部位, value) => `${部位}认知开发度 +${value}`,
  },
  道德下降: {
    纯爱: () => `她对你的信任增加了。`,
    真相: (value) => `道德底线 -${value}（越来越能接受禁忌行为）`,
  },
  境界提升: {
    纯爱: (旧, 新) => `关系提升：${旧} → ${新}`,
    真相: (旧, 新) => `境界提升：${旧} → ${新}`,
  },
  丈夫打断: {
    纯爱: () => `赵霞突然想起了一些事情...`,
    真相: () => `【丈夫打断】苏文突然回家，行为被迫中断。`,
  },
};
```

### `getSystemHint(type, data, ...args)`

获取系统提示文本，根据当前模式返回对应版本。

## 使用示例

```typescript
// 获取依存度显示名
const 依存度显示名 = getDisplayText('依存度', data);
// 纯爱模式返回：'好感度'
// 真相模式返回：'对主角依存度'

// 获取境界名称
const 境界名 = getRealmName(3, data);
// 纯爱模式返回：'信任'
// 真相模式返回：'溺深'

// 获取系统提示
const 提示 = getSystemHint('依存度提升', data, 10);
// 纯爱模式返回：'赵霞对你的好感提升了！(+10)'
// 真相模式返回：'赵霞对你的依存度提升，她越来越需要你。(+10)'
```
