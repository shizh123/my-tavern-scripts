# 数据保护系统

> 创建日期：2026-01-17
> 文件位置：`src/赵霞/脚本/游戏逻辑/dataProtection.ts`
> 状态：已实现

## 概述

数据保护系统防止 AI 篡改关键游戏数据。通过在时间推进后创建数据快照，然后验证 AI 回复后的数据，自动还原被篡改的字段。

## 核心设计理念

1. **快照机制**：在脚本处理数据后创建快照
2. **篡改检测**：对比 AI 处理后的数据与快照
3. **自动还原**：检测到篡改时自动回滚
4. **白名单机制**：允许 AI 更新的字段不受保护

## 核心函数

### `initDataProtection()`

初始化数据保护系统。

**调用时机**：游戏逻辑模块加载时

---

### `createDataSnapshot(data)`

创建数据快照。

**调用时机**：时间推进之后、AI 处理之前

**重要**：必须在时间推进之后创建快照，否则时间更新会被误认为篡改

---

### `updateSnapshotValue(path, value)`

更新快照中的特定值。

**用途**：当脚本合法修改某个字段时，同步更新快照

**示例**：
```typescript
data.世界.游戏阶段 = '梦境';
updateSnapshotValue('世界.游戏阶段', '梦境');
```

---

### `validateAndRestoreData(data)`

验证并还原被篡改的数据。

**调用时机**：AI 回复处理完成后

**返回值**：
```typescript
{
  detected: boolean;           // 是否检测到篡改
  tamperedFields: string[];    // 被篡改的字段列表
  restoredValues: Record<string, any>;  // 还原的值
}
```

---

### `generateProtectionReport(result)`

生成保护报告（调试用）。

---

## 工作流程

```
1. 时间推进 (TimeSystem.advance)
      ↓
2. 创建快照 (createDataSnapshot)
      ↓
3. AI 处理回复
      ↓
4. 验证并还原 (validateAndRestoreData)
      ↓
5. 写入数据
```

## 与其他系统的交互
