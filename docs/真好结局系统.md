# 真好结局系统

> 创建日期：2026-01-17
> 文件位置：`src/赵霞/脚本/游戏逻辑/trueEndingSystem.ts`
> 状态：已实现

## 概述

真好结局系统处理游戏中的两种最佳结局：**真好结局**和**完美真爱结局**。两种结局共用同一套10阶段剧情流程，但在叙事风格和赵霞的状态表现上有所不同。

---

## 两种结局的区别

| 对比项 | 真好结局 | 完美真爱结局 |
|--------|----------|--------------|
| **结局标题** | 【TRUE END】禁忌之爱 | 【PERFECT TRUE END】命中注定 |
| **触发条件** | 5场景全完成+全正确 | 真好结局条件 + 记忆连贯性=3 |
| **赵霞状态** | 被改造、被洗脑 | 命中注定、发自内心 |
| **叙事基调** | "她的记忆被改写" | "她从一开始就在等待" |
| **开场风格** | 满足感 | 发自内心的笑意 |
| **情感表达** | 主动但带有被控制感 | 真挚的"终于等到你" |

---

## 触发条件

### 真好结局（禁忌之爱）

| 条件 | 要求 |
|------|------|
| 时间 | Day 5, 20:00+ |
| 循环状态 | 结局判定 或 已破解 |
| 场景完成 | 5个场景全部完成 |
| 正确场景数 | = 5（全部正确） |
| 记忆连贯性 | < 3 |

### 完美真爱结局（命中注定）

| 条件 | 要求 |
|------|------|
| 时间 | Day 5, 20:00+ |
| 循环状态 | 结局判定 或 已破解 |
| 场景完成 | 5个场景全部完成 |
| 正确场景数 | = 5（全部正确） |
| 记忆连贯性 | = 3（完美记忆路线） |

**完美记忆路线**：玩家在进入场景5之前，按顺序完成了场景1→2→3。

---

## 结局判定代码

```typescript
// 真好结局/完美真爱结局：完成所有5个场景且全部正确
if (全部完成 && 正确场景数 === 5) {
  const 记忆连贯性 = getScene5LockedCoherence(data);
  const 是完美记忆路线 = 记忆连贯性 === 3;

  if (是完美记忆路线) {
    data.结局数据.当前结局 = '完美真爱结局';
    data.结局数据.是完美记忆路线 = true;
  } else {
    data.结局数据.当前结局 = '真好结局';
    data.结局数据.是完美记忆路线 = false;
  }
}
```

## 10阶段剧情设计

### 阶段0：序幕（纪念日晚餐）

- **时间**：20:00
- **场景**：结婚纪念日晚餐
- **锚点**：纪念日提及、餐桌布置
- **最少轮数**：1轮
- **赵霞状态**：表面平静，内心期待，酒里已下药

### 阶段1：午餐（最后的敬酒）

- **场景**：苏文入座，三人用餐
- **关键**：赵霞给苏文倒下药的酒
- **锚点**：苏文入座、敬酒、苏文饮酒
- **最少轮数**：2轮

### 阶段2：药效（沉睡的丈夫）

- **场景**：安眠药发作，苏文昏倒
- **关键**：赵霞露出真实面目
- **锚点**：苏文犯困、苏文昏倒、赵霞转变
- **最少轮数**：2轮

### 阶段3：准备（布置舞台）

- **场景**：将苏文绑在床上，架设摄像机
- **关键**：准备"仪式"
- **锚点**：绑缚苏文、架设摄像机
- **最少轮数**：2轮

### 阶段4：展示（改造的身体）

- **场景**：赵霞展示被改造的身体
- **内容**：情趣内衣、淫纹刺青、乳环等
- **锚点**：脱下外衣、展示改造
- **最少轮数**：2轮

### 阶段5：告白（镜头前的真相）

- **场景**：赵霞面对摄像机自白
- **内容**：揭露所有第一次的真相
- **锚点**：自我介绍、揭露真相、第一次归属
- **最少轮数**：3轮

### 阶段6：表白（禁忌的告白）

- **场景**：赵霞向玩家正式表白
- **内容**：深情告白，完全交付
- **锚点**：转向玩家、深情表白
- **最少轮数**：2轮

### 阶段7：仪式（婚床上的背叛）

- **场景**：在婚床上结合
- **内容**：当着昏迷苏文的面，羞辱与结合
- **锚点**：上床、羞辱苏文、结合
- **最少轮数**：3轮

### 阶段8：种子（生命的请求）

- **场景**：赵霞请求内射
- **内容**：想要怀上儿子的孩子
- **锚点**：请求内射、想要孩子
- **最少轮数**：2轮

### 阶段9：完成（新的开始）

- **场景**：事后温存，结局达成
- **内容**：赵霞摘下戒指交给玩家
- **锚点**：事后温存、戒指交付、结局宣言
- **最少轮数**：1轮

## 阶段推进逻辑

```typescript
export function shouldAdvancePhase(state: TrueEndingState): boolean {
  const phaseConfig = TRUE_ENDING_PHASES[state.currentPhase];

  // 条件1：最少轮数满足
  if (state.turnsInPhase < phaseConfig.minTurns) {
    return false;
  }

  // 条件2：所有锚点事件完成
  const allAnchorsComplete = phaseConfig.anchorEvents.every(
    anchor => state.completedAnchors.includes(anchor)
  );

  return allAnchorsComplete;
}
```

## 状态数据结构

```typescript
export interface TrueEndingState {
  isActive: boolean;        // 是否已激活
  currentPhase: number;     // 当前阶段 (0-9)
  turnsInPhase: number;     // 当前阶段对话轮数
  completedAnchors: string[]; // 已完成锚点
  totalTurns: number;       // 总对话轮数
  isComplete: boolean;      // 是否已完成
}
```

## 核心函数

### `shouldActivateTrueEnding(data)`

检测是否应该激活真好结局流程（同时支持真好结局和完美真爱结局）。

### `isPerfectTrueEnding(data)`

检测是否满足完美真爱结局条件（记忆连贯性=3）。

### `generateTrueEndingPrompt(state, userInput)`

生成当前阶段的AI引导Prompt。

### `processTurnEnd(state, aiResponse)`

处理回合结束，更新状态，检测阶段推进。

### `generateTrueEndingEntryReplacement(data)`

生成入口替换消息（首次激活时），根据是否为完美真爱结局生成不同开场。

### `generateTrueEndingComplete(data)`

生成结局完成消息，根据结局类型返回不同文本。

## 完美真爱结局特殊处理

当检测到完美真爱结局时，系统会进行以下特殊处理：

### AI引导补充

```typescript
【完美记忆路线】
玩家在进入场景5之前，按顺序完成了场景1-2-3。
赵霞拥有完整的记忆重构，她对玩家的感情不是被洗脑，而是"命中注定"。

【AI特别指引】
完美记忆路线的赵霞不是"被改造"的，而是"从一开始就在等待"的。
她的选择是发自内心的，不是被催眠的。
描写她时，要体现出"终于等到你"的感动。
```

### 开场文本差异

**真好结局开场**：
```
暮色透过窗帘洒进餐厅...嘴角挂着一抹淡淡的笑意。
```

**完美真爱结局开场**：
```
暮色温柔地洒进餐厅...嘴角挂着一抹发自内心的笑意。
那是一种终于等到了什么的满足感。
```

## 结局完成后

- 标记：时间循环被打破
- 标记：真好结局 或 完美真爱结局 达成
- 解锁：后日谈模式
- 设置：`data.结局数据.后日谈已解锁 = true`
- 设置：`data.世界.循环状态 = '已破解'`
