> 创建日期：2026-01-17

> 状态：已实现

## 概述

混乱结局是场景5专属的坏结局，当玩家"明知故犯"触发不合适的行为时触发。设计理念是惩罚作死玩家，正常按剧情走的玩家不会触发。

**核心设计**（2026-01-19 重新设计）：
- 混乱结局**只在场景5中触发**
- 触发后**立即锁死游戏**，玩家无法进行任何操作
- 不再有"延迟触发"或"前兆提示"机制

---

## 触发条件

### 1. 性行为检测

| 路线类型 | 容错次数 | 混乱度变化 | 说明 |
|---------|---------|-----------|------|
| 非完美路线（连贯性<3） | 1次 | 80→90→100 | 第1次警告+10，第2次触发 |
| 完美路线（连贯性=3） | 2次 | 80→87→94→101 | 第1、2次警告+7，第3次触发 |

**设计理念**：每次犯错混乱值会明显上涨，玩家看到数值接近100就会明白再犯错就要触发坏结局了。

### 2. 打断婚礼仪式

- 在步骤3-5期间检测
- 直接触发混乱结局，无容错

---

## 触发时机

**场景5直接触发并锁死**（2026-01-19 重新设计）：

```
场景5中性行为/打断婚礼
     ↓
检测到违规行为（promptInjection.ts）
     ↓
混乱度达到100
     ↓
markConfusionEnding() - 直接设置 已触发=true
     ↓
applyConfusionEndingState() - 锁定游戏状态
     ↓
返回结局消息，游戏彻底结束
```

**关键点**：
- 触发后游戏立即锁死，玩家无法继续操作
- 每次输入都会显示"游戏已结束"的锁定消息
- 不再有"日常阶段触发"或"前兆提示"

> **历史设计**（已废弃）：
> - v1（已废弃）：延迟1小时触发
> - v2（已废弃）：退出场景5后在日常阶段触发
> - v3（当前）：场景5中直接触发并锁死

## 核心函数

### `detectSexualBehavior(userInput)`

检测玩家输入是否包含性行为关键词。

### `detectWeddingInterruption(userInput)`

检测玩家输入是否打断婚礼仪式。

### `checkSexualBehaviorInScene5(data, userInput)`

检测并处理场景5中的性行为，返回：
- 是否检测到
- 是否需要警告
- 是否需要标记混乱
- 警告消息

### `markConfusionEnding(data, reason)`

标记并立即触发混乱结局（2026-01-19 重新设计）：

- 直接设置 `已触发=true`（不再有"标记"和"触发"两步）
- 记录触发原因
- 游戏立即锁死

### `checkConfusionEnding(data, forceFirstTrigger)`

检测混乱结局状态：

- `已触发=true`：返回锁定消息
- `forceFirstTrigger=true`：返回首次触发模板（用于刚触发的情况）
- `forceFirstTrigger=false`：返回持续锁定模板（用于后续检测）
- 未触发：返回空

## 游戏锁定机制

```typescript
export function canEnterDreamForConfusion(data: SchemaType): boolean {
  // 如果混乱结局已触发，不能进入任何梦境
  if (data.梦境数据.混乱结局?.已触发) {
    return false;
  }
  return true;
}
```

**注意**：2026-01-19 重新设计后，不再有"前兆提示"机制。混乱结局在场景5中直接触发并锁死，没有过渡阶段。

## 优先级

**混乱结局 > 发现结局**

原因：触发混乱结局时，通常丈夫已被"处理"，发现结局没有意义。

## 玩家路线对比

| 玩家类型 | 行为 | 结果 |
|---------|------|------|
| 正常玩家 | 按12步剧情走 | ✅ 安全 |
| 完美玩家 | 完成1-2-3 + 场景5正常 | ✅ 很安全 + 最佳结局 |
| 性急玩家（无场景3） | 场景5尝试2次性行为 | ❌ 混乱结局 |
| 性急玩家（有场景3） | 场景5尝试3次性行为 | ❌ 混乱结局 |
| 破坏玩家 | 打断婚礼仪式 | ❌ 直接混乱结局 |
