# 时间系统

> 创建日期：2026-01-17
> 文件位置：`src/赵霞/脚本/游戏逻辑/timeSystem.ts`
> 状态：已实现

## 概述

时间系统是赵霞项目的核心系统之一，采用"零BUG设计"原则，确保时间推进的一致性和可靠性。

## 核心设计原则

1. **单一入口**：所有时间修改通过 `TimeSystem.advance()` 函数
2. **单一数据源**：所有时间读取从 `data.世界.时间`
3. **原子更新**：时间、时间戳、状态栏刷新标记一起更新
4. **强制刷新**：每次时间更新后标记 `状态栏需要刷新 = true`
5. **楼层追踪**：记录每个楼层的时间更新

---

## 数据结构

```typescript
世界: {
  当前天数: number;        // 1-5天
  当前小时: number;        // 0-23小时
  时间: string;           // 格式："Day 1, 08:00"
  时间戳: number;         // Date.now()，用于同步验证
  状态栏需要刷新: boolean; // 关键标记
  循环状态: '进行中' | '结局判定' | '已破解';
}
```

---

## 核心函数

### `TimeSystem.advance(data, hours)`

推进时间的唯一入口。

**参数**：
- `data`: 游戏数据对象
- `hours`: 推进小时数（默认1小时）

**行为**：
1. 计算新时间（处理跨天）
2. 检查是否触发结局判定（Day 5, 20:00）
3. 原子更新所有时间相关变量
4. 标记状态栏刷新

**返回值**：
```typescript
{
  success: boolean;
  oldTime: string;
  newTime: string;
  day: number;
  hour: number;
}
```

---

### `TimeSystem.isDreamWindowOpen(data)`

检查是否在梦境入口窗口。

**时间范围**：22:00-01:59（含跨天）

**用途**：场景1-4的梦境入口检测

---

### `TimeSystem.isWakeUpTime(data)`

检查是否到达醒来时间。

**时间点**：10:00

**用途**：场景1-4梦境结束，切换回日常阶段

---

### `TimeSystem.isEndingTime(data)`

检查是否到达结局判定时间。

**时间点**：Day 5, 20:00+

**历史变更**：2026-01-16 从 12:00 调整为 20:00，给完美记忆路线足够时间

---

### `TimeSystem.getHoursUntilReset(data)`

计算距离结局判定还剩多少小时。

**计算方式**：
- 结局判定点：Day 5, 20:00 = (5-1)*24 + 20 = 116 小时
- 剩余 = 116 - 当前总小时数

---

### `TimeSystem.processTimeSkipRequest(data, userInput)`

处理时间跳过请求。

**设计决策**：时间跳过功能已禁用

**行为**：
1. 检测跳过意图（睡觉、跳到特定时间、下一天）
2. 返回阻止结果和MVU提示消息

## 时间推进规则

### 每轮推进

- 每次 `MESSAGE_RECEIVED` 事件推进1小时
- `GENERATION_ENDED` 事件不推进（防止重复）
- `MESSAGE_SWIPED`（ROLL）不推进

### 跨天处理

```typescript
while (totalHours >= 24) {
  totalHours -= 24;
  day += 1;
}
if (day > 5) {
  day = 1; // 循环重置
}
```

### 结局触发

- Day 5, 20:00 自动触发 `循环状态 = '结局判定'`

## 特殊时间点

| 时间点 | 事件 |
|--------|------|
| 08:00-19:00 | 场景5入口时间（安眠药触发） |
| 10:00 | 场景1-4梦境结束，赵霞醒来 |
| 19:00 | 场景5触发退出描写 |
| 20:00 | 场景5强制退出 / Day 5结局判定 |
| 22:00-01:59 | 场景1-4梦境入口窗口 |

---

## 与其他系统的交互

1. **状态栏**：时间更新后标记 `状态栏需要刷新`
2. **梦境系统**：检测梦境入口窗口和醒来时间
3. **结局系统**：触发结局判定
4. **数据保护**：时间更新后创建快照
