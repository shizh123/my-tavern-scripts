# 双模式外观系统

> 创建日期：2026-01-17
> 文件位置：`src/赵霞/脚本/游戏逻辑/appearanceSystem.ts`
> 状态：已实现

## 概述

双模式外观系统实现了游戏的核心双重叙事机制。根据是否进入过梦境，系统在两套完全不同的外观配置之间切换。

## 核心设计

### 纯爱模式（伪装）
- 触发条件：`世界.已进入过梦境 = false`
- 风格：温柔贤淑、保守得体
- 玩家视角：以为在玩正常恋爱游戏

### 真相模式（真实）
- 触发条件：`世界.已进入过梦境 = true`
- 风格：随境界递增的暴露程度
- 玩家视角：发现真相后的堕落过程

## 核心函数

### `getRealmConstraints(realm, 已进入过梦境)`

获取当前境界的外观约束。

**返回值**：
```typescript
{
  暴露程度范围: ExposureLevel[];
  最低暴露程度: ExposureLevel;
  妆容浓度范围: MakeupLevel[];
  最低妆容浓度: MakeupLevel;
  允许特殊装饰: boolean;  // 项圈、手铐等
  允许身体标记: boolean;  // 纹身、烙印等
}
```

---

### `getStyleGuidance(realm, 已进入过梦境)`

获取风格指导（给 AI 的建议）。

**返回值**：
```typescript
{
  整体风格: string;
  服装建议: string;
  气质关键词: string[];
  禁忌元素?: string[];
  特殊说明?: string;  // 境界5专属
}
```

---

### `generateAppearanceConstraintPrompt(data)`

生成外观约束提示词（注入给 AI）。

**自动处理**：
- 模式切换（纯爱/真相）
- 境界5苏文在场时的表面保守
- 睡眠时间的睡衣提示

---

### `generateDetailedInteractionPrompt(data)`

生成详细的互动范围提示词。

**纯爱模式**：简化显示，有境界限制

**真相模式**：
- 梦境中：强调无限制开发
- 现实中：显示各部位详细状态

## 真相模式数值计算

### 依存度
```typescript
依存度 = 所有部位进度的平均值
```

### 道德底线
```typescript
道德底线 = 100 - 依存度
```

### 境界
```typescript
境界 = Math.ceil(依存度 / 20)
```
