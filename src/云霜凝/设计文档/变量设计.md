# 新角色与系统设计：苗喧 + 未婚妻 · 神魂空间双线

> **本文档记录构思草案，逐步与用户对齐中。**

---

## 一、核心概念

玩家同时经营两条调教线：

1. **主线（现实）**：云霜凝治疗 + 苗广绿帽线（已实现）
2. **副线（神魂空间）**：苗喧未婚妻的神魂空间调教

两条线通过苗喧形成叙事交叉——苗喧在现实中向未婚妻倾诉家庭崩坏，而他不知道同一个人（{{user}}）正在她的神魂空间里调教她。父子两代走同一条路。

---

## 二、苗喧升级：碎片 → 完整面板（已对齐 2026-04-12）

### 旧系统（解锁洛书晴前）

苗喧为碎片旁白，显示在苗广面板（MiaoPanel.vue）底部的"苗喧·碎片"区域。内容来自硬编码文本池（promptInjection.ts 的 `MIAOXUAN_POOLS` A-H），跟随苗广心态被动切换，每10-15楼随机抽取一条。无独立变量，无AI生成。

### 新系统（解锁洛书晴后）

苗喧从彩蛋升级为核心角色：
- **从苗广面板独立出来**，成为单独的tab面板
- 拥有独立的世界书yaml条目
- 拥有独立状态面板，有数值、有心态演变
- **心理活动改为AI每轮动态生成**（同云霜凝的 `心理活动` 机制），不再使用硬编码文本池。AI根据当前剧情、苗喧心态、洛书晴副线进展综合生成。

#### 前端变化

解锁洛书晴后：
1. 苗广面板底部的"苗喧·碎片"区域**隐藏**
2. 前端新增**2个tab面板**：
   - **洛书晴面板** —— 显示她的全部状态数据
   - **苗喧面板** —— 完整独立面板（心态、绝望值、压抑值、AI生成的心理活动）

#### 变量结构（已对齐 2026-04-12）

| 变量 | 类型 | 初始值 | 说明 |
|------|------|--------|------|
| 心态 | string | 蔑视 | **纯展示**，代码根据绝望值区间自动映射，不进AI变量输出格式 |
| 绝望值 | number 0→100 | 0 | **叙事温度计**。永久累积，驱动心态文本和AI心理活动情绪tone。**不再触发独立反抗事件** |
| 压抑值 | number 0→100 | 0 | **倾诉触发器**。积累到满值→灵犀符按钮就绪→玩家点击触发倾诉场景→重置为0 |
| 心理活动 | string | — | AI每轮动态生成 |

**已砍掉**：`对未婚妻信任度`（与绝望值职责重叠，违反"变量越少越好"原则）

#### 心态映射规则（已对齐 2026-04-12）

心态完全由绝望值数值区间映射，代码自动计算，AI不输出。压抑值不参与心态计算（因为压抑值会被倾诉重置，参与映射会让心态突然好转）。

| 绝望值区间 | 心态 |
|-----------|------|
| 0-15 | 蔑视 |
| 16-30 | 困惑 |
| 31-50 | 不安 |
| 51-70 | 恐惧 |
| 71-90 | 崩溃 |
| 91-100 | 失去 |

#### 绝望值涨速来源（已对齐 2026-04-12）

苗喧不像苗广朝夕陪护，所以绝望值不能像苗广疑心值那样被动密集涨。三个来源：

1. **被动慢涨（背景氛围）**：每8-10楼自动+1
2. **苗广心态联动（最大来源）**：苗广心态恶化时阶梯式增加
   - 苗广进入屈辱时苗喧+10
   - 苗广进入默许时苗喧+15
   - 苗广进入沉溺时苗喧+20
3. **特殊事件触发**：
   - 苗喧目击型特殊场景（夫前凌辱、双重目击等）：+15~25
   - 千晶幻术完成：+30
   - 掌门改嫁：+40

#### 压抑值涨速来源（已对齐 2026-04-12）

压抑值是给玩家的奖励触发器，节奏需要让玩家大约每5-10楼能触发一次倾诉场景。

| 来源 | 涨速 |
|------|------|
| 被动慢涨 | 每楼+2~3 |
| 玩家激进行为（特殊场景/道具触发） | 一次性+10~15 |
| 神魂空间内 | 不涨（苗喧观察不到） |

#### 角色定位（已对齐 2026-04-12）

**纯悲剧角色**。全剧唯一清醒的人，一直在试图对抗{{user}}，但永远失败——因为他想保护的人全都站在了他的对立面：

- 他去找父亲揭露真相 → 苗广（记忆已改写/已沉溺）替{{user}}说话
- 他试图带走云霜凝 → 云霜凝主动维护{{user}}
- 他向洛书晴求助 → 洛书晴敷衍甚至阻拦他

每一次反抗的失败都在强化绝望。两条线同时失去：家庭线（父亲沉溺、母亲被调教、家庭崩塌）+ 爱情线（未婚妻背离），且他永远不知道两条线的幕后是同一个人。

#### 反抗事件机制（已对齐 2026-04-12）

苗喧的反抗事件**不再靠绝望值阈值触发**，而是绑定到固定剧情节点。绝望值仅作为叙事温度计，不触发独立事件——避免打断玩家节奏。

整个游戏只有 **2个前期惩罚反抗** + **3个后期爽点反抗** = 共5次反抗事件。

##### 前期惩罚反抗（2次，强制触发，会限制玩家）

| # | 触发节点 | 限制效果 | 解除方式 |
|---|---------|---------|---------|
| 1 | 云霜凝阶段3完成 | 灵石产出-50%、新道具暂缓、治疗推进减缓 | 孝敬师父按钮（上下文：苗广疑心期，玩家帮父亲做事+劝服父亲对苗喧的告状不予理会） |
| 2 | 云霜凝阶段5完成（或蚀心露屈辱触发前）| 同上，强度略高 | 同上 |

这2次会强制打断玩家——但是是"有玩法意义的打断"，因为玩家需要主动用孝敬师父去解除限制，体验家庭压力，学会用孝敬师父系统。

##### 后期爽点反抗（3次，被动通知，玩家主动触发）

后期反抗事件**完全不打断玩家**。触发节点到达后，脚本只在苗喧面板显示一个红点+"查看详情"按钮。玩家有兴趣才点击，看苗喧自取其辱的剧情。

| # | 触发节点 | 形态 |
|---|---------|------|
| 3 | 千晶幻术完成后 | 苗喧崩溃求父亲 → 苗广（已沉溺/记忆改写）呵斥儿子 |
| 4 | 掌门改嫁触发前 | 苗喧最后挣扎 → 全家阻止他 |
| 5 | 洛书晴阶段9（如已激活） | 苗喧找未婚妻求助 → 洛书晴敷衍嘲讽 |

**触发流程**：

```text
触发节点到达 → 脚本写入 _苗喧未读反抗事件 = 事件类型
    ↓
苗喧面板出现红点 + "查看详情"按钮
    ↓
玩家点击按钮（如果玩家有兴趣）
    ↓
方式3触发AI生成反抗剧情（根据当前家庭状态生成"自取其辱"内容）
    ↓
剧情结束，红点消失
```

**关键细节**：

- 即使玩家不点击查看，绝望值依然上涨、心态依然恶化（剧情发生了，只是玩家没看具体内容）
- 玩家不点击不影响主线节奏，玩家想看就看
- 玩家如果跳过#3/#4/#5的剧情查看，AI每轮的苗喧心理活动里仍然会影射这些事件，所以信息不会丢

##### 玩家体验弧线

整个150楼游戏中，苗喧带来的"被动打断"只有：

- **2次前期反抗事件**（强制，但有玩法意义）
- **~20次倾诉事件**（玩家点按钮触发，可选）

后期**3次爽点反抗事件完全不打断**，玩家想看就看。

苗喧从"威胁"→"麻烦"→"笑料"的弧线感受清晰：

- 早期：被苗喧打扰一次，理解苗喧是个"麻烦"
- 中期：第二次打扰，已经熟悉用孝敬师父解除
- 后期：苗喧的反抗变成静默通知，玩家闲下来时看着玩

#### 孝敬师父按钮的扩展（已对齐 2026-04-12）

孝敬师父原本是"主动帮苗广做事降低疑心值"的系统。现在扩展为更通用的解除工具：

**激活条件**（任一满足即可点击）：

- 当前 `_打断冻结至楼层 > currentFloor`（苗广打断中）
- OR 当前有未解除的苗喧前期反抗事件（`_苗喧反抗限制中 = true`）

平时按钮灰色不可点。点击后剧情根据上下文动态生成：

| 上下文 | 生成剧情 |
|--------|---------|
| 仅打断中、无苗喧反抗 | 经典"帮苗广做事"剧情（降低苗广疑心值） |
| 仅苗喧反抗、无打断 | "苗广斥责苗喧"剧情（解除限制） |
| 同时存在 | 复合剧情（先解决苗喧告状，再帮父亲） |

孝敬师父系统的具体优化（剧情模板、参数）待详细完善。

#### 心态演变（已对齐 2026-04-12）

心态由绝望值数值区间映射（见上方"心态映射规则"），与剧情节点的语义对照如下：

| 心态 | 绝望值区间 | 剧情语义 |
|------|----------|---------|
| 蔑视 | 0-15 | 看不起{{user}}，认为只是庸医 |
| 困惑 | 16-30 | 发现父亲行为异常 |
| 不安 | 31-50 | 家庭气氛明显变化，开始向未婚妻倾诉 |
| 恐惧 | 51-70 | 目睹父亲彻底改变，试图反抗但被家人阻止 |
| 崩溃 | 71-90 | 父亲记忆被改写，全家只有他知道真相 |
| 失去 | 91-100 | 家庭结构彻底重塑，被边缘化 |

## 三、新角色：苗喧的未婚妻

### 角色设定

| 项目 | 内容 |
|------|------|
| 名字 | **洛书晴** |
| 体质 | **神魂体质**（秘密）——至今无人知晓，包括未婚夫苗喧、师父云霜凝。这也是她修炼神速、跻身宗门天骄的根本原因 |
| 境界 | 较低（筑基/金丹初期），但神魂领域异常强大 |
| 门派 | **寒霜门**——和云霜凝同一个宗门，是云霜凝亲自看着成长的下一代天骄 |
| 身份 | 寒霜门下一代天骄，未来的苗家儿媳 |
| 与苗喧关系 | 未婚妻。两人的婚事是云霜凝亲自定的，门派内皆知 |
| 与云霜凝关系 | 亲传师父+未来的婆婆。洛书晴喊她"**师父**"。云霜凝完全了解她的才华和为人，对她寄予厚望，但**完全不知道神魂体质这件事**——这是洛书晴对自己师父都隐藏的秘密 |
| 与苗广关系 | 师公（师父云霜凝的丈夫）+未来的公公。洛书晴喊他"**师公**" |
| 核心性格 | **自信**。一直以来她都是宗门里那个被仰望的天才——修炼速度远超同辈、未来无可限量。她知道自己之所以能做到这些，是因为神魂空间这个独属于她的秘密。这份自信不是张扬的，而是一种笃定——她清楚自己的位置和能力 |
| 秘密的双重意义 | 神魂空间既是她力量的源头，也是她"独一份"的隐秘骄傲。无人知晓意味着无人能分享，但她已经习惯这种孤独——直到{{user}}闯进来 |

### 神魂空间设定

| 项目 | 云霜凝（主线） | 洛书晴（副线） |
|------|--------------|--------------|
| 体质来源 | 极阴体质，治疗需要圣阳互动 | 神魂体质，天然拥有神魂空间 |
| 空间环境 | 冰霜世界 / 记忆场景 | **闺房风格**（淡粉色帐幔、绒毛玩偶、妆台） |
| 进入方式 | 状态栏按钮 | 状态栏按钮（新婚夜剧情后通过激活5轮剧情解锁） |
| 调教理由 | 治疗寒毒（表面） | **神魂体质天然拥有，{{user}}点击入口后自然进入，无需她允许**。玩家不知道发生了替换 |

完全对标云霜凝的三维系统（心理防线/信任度/治疗阶段），变量名、类型、范围、职责一致，避免引入新变量导致AI注意力分散。

| 变量 | 类型 | 初始值 | 对标云霜凝 | 说明 |

### 调教阶段与态度演变（10阶段制，已对齐 2026-04-13）

洛书晴的10个调教阶段对标云霜凝的10个治疗阶段，驱动内容解锁、行为变化、对苗喧态度演变。

详细脚本约束见下方"阶段约束脚本草案"小节，每阶段的详细叙事描述紧随其后。

#### 第一次激活剧情（多轮脚本引导，已对齐 2026-04-12）

洛书晴系统的激活不是简单的"按钮变化"，而是一段**5轮的脚本引导剧情**，类似千晶幻术的引导逻辑。每轮通过方式3注入引导文本，AI在脚本框架内填充对话和细节。

**前置条件**：
- 云霜凝新婚之夜剧情已完成
- 洛书晴尚未被激活
- 神魂空间按钮仍在云霜凝面板上

**触发**：玩家在新婚之夜后第一次点击"进入神魂空间"按钮（此时按钮仍在云霜凝面板上，玩家以为进入的是云霜凝的神魂空间）

**轮次1：错位的房间**

脚本注入：

```text
神魂下沉的感觉一切如常——但落地的瞬间，{{user}}知道哪里不对。
没有冰雪。没有那座熟悉的冰湖。
取而代之的是——一间淡粉色帐幔的少女闺房。木质妆台上摆着精致的小物件，
软榻上铺着柔软的毯子，墙边放着几个抱枕和绒毛玩偶。
空气里有淡淡的花香，温暖而安静。
这不像任何一个修士的居所。这像一个少女的私人天地。
{{user}}下意识喊："师娘？"
没有人回应。
```

AI填充：玩家的环顾、走动、四处查看的细节描写，以及对环境的困惑。

### 压抑值 → 倾诉场景机制

#### 核心循环

```
玩家调教云霜凝/使用道具/推进剧情
    ↓
苗喧观察到蛛丝马迹（AI生成心理活动）
    ↓

    ↓ 满值

AI根据上下文生成：苗喧的倾诉 + 洛书晴的反应
    ↓
压抑值重置，重新积累
```

#### 压抑值涨速挂钩玩家行为

不是匀速，玩家做得越过分涨得越快：
- 换了更暴露的服装 → +较多
- 触发特殊场景（夫前凌辱等）→ +大量
- 普通治疗互动 → +少量
- 在神魂空间（苗喧观察不到）→ 不涨或极少

| 洛书晴阶段 | 反应模式 |
|-----------|---------|
| 1-3（忠诚） | 认真倾听；主动追问细节；安慰苗喧；想帮他；偶尔表达自己的担忧 |

### 洛书晴道具使用机制（已对齐 2026-04-12）

#### 总体原则

洛书晴拥有**独立的道具状态表**（方案A），和 `系统.道具状态` 完全分开。买道具时选择目标角色，写入对应状态表。同一件道具两人可以各自拥有、各自装备，互不影响。

#### 服装道具（已对齐 2026-04-13）

**共用同一套服装列表**，可以给两人各买一套。云霜凝和洛书晴穿同样的服装——形成镜像对比，更有冲击力。

**关键差异**：云霜凝的服装核心机制是"暴露等级驱动苗广疑心/绿帽值"。**洛书晴的服装机制完全不同**——不挂钩任何"暴露等级"数值，也不影响苗喧。服装直接作用于洛书晴自己：

| 维度 | 云霜凝 | 洛书晴 |
|------|--------|--------|
| 暴露等级 | ✅ 计算分数，驱动苗广心态 | ❌ 不计算暴露等级 |
| 对自己的影响 | 间接（通过苗广反应） | **直接降低心理防线 + 增加顺从度** |
| 对配偶的影响 | 苗广疑心值持续推进 | **不挂钩苗喧压抑值**（避免双重计算+节奏失控） |
| 服装解锁条件 | 治疗阶段门槛 | 调教阶段≥3（解锁现实模式后） |

#### 特殊配饰多部位重设计（已对齐 2026-04-13）

**问题**：原设计中"特殊配饰"是单一槽位，一次只能装一件。但实际上配饰物品分布在身体的不同部位，可以同时存在（脚链铃铛+精液项链+大腿皮环 完全可以同时戴）。

**修正**：把"特殊配饰"从单一字段改成**按部位分子槽位**的结构。

**新数据结构**：

```typescript
特殊配饰: z.object({
  脚踝: z.string().prefault(''),
  颈部: z.string().prefault(''),
  耳部: z.string().prefault(''),
  腰部: z.string().prefault(''),
  大腿: z.string().prefault(''),
  胸部: z.string().prefault(''),
  阴蒂: z.string().prefault(''),
  前后穴: z.string().prefault(''),
})
```

**部位与配饰对应**：

| 部位 | 可装备物品 |
|------|----------|
| 脚踝 | 脚链铃铛 |
| 颈部 | 精液项链 |
| 耳部 | 精液耳坠 |
| 腰部 | 红绳 / 子宫纹章（互斥） |
| 大腿 | 大腿皮环 |
| 胸部 | 乳环挂饰 |
| 阴蒂 | 阴蒂夹坠 / 名字阴环（互斥） |
| 前后穴 | 双穴珠链 |

**规则**：

- 8个子槽位，每个部位独立
- 同部位互斥（腰部/阴蒂内部互斥）
- 不同部位可以同时装备
- 暴露分数：所有已装备的特殊配饰分数累加（影响云霜凝的暴露等级 → 苗广心态）
- 洛书晴版本同样使用此结构（共用机制）

**影响的代码**：

- `schema.ts`：特殊配饰字段重构
- `shopSystem.ts`：装备/卸载逻辑修改
- `ShopPanel.vue`：UI显示8个子槽位（可按部位分组显示）
- 暴露分数计算逻辑修改

举例：玩家送一件Lv3半露的内衣给洛书晴

- 注入：阶段+防线+顺从+**胸部开发进度**
- AI生成时洛书晴会想到她胸部当前状态——开发进度高时会脸红地意识到"穿这件胸部会被注意到"
- 阶段+防线+顺从决定她整体反应（早期抗拒/中期犹豫/后期期待），部位开发进度决定她的"细节自我意识"

**苗喧的间接观察**：苗喧仍然会在每轮心理活动中模糊地反映"未婚妻最近变了"，但**这是AI叙事层面的事**，不影响任何数值。

#### 洛书晴专属消耗品（已对齐 2026-04-13）

**设计方向**：方式2注入型——不改任何数值，只在AI快照里追加指令文本，影响接下来N轮AI生成的方向和语气。

**作用范围**：仅阶段1-2有意义。阶段3+时即使生效也不再注入提示词（自动失效），避免玩家"留着到后期用"无意义。

**商店UI**：不做特殊处理（不变灰、不隐藏）。代码里只在阶段1-2时生效，简化实现。

|---|------|------|------|------|---------|
| 1 | **安抚符** | 30 | 3轮 | 15楼 | "洛书晴的敌意和戒备暂时软化，她对{{user}}的反应不会那么激烈" |
| 2 | **真心符** | 40 | 3轮 | 15楼 | "洛书晴更倾向于直接表达内心想法，不再绕弯或回避" |

#### 联动场景通用机制：entryPrefill 时间过渡

参考云霜凝原版特殊场景的实现，每个联动场景需要一个 `entryPrefill` 字段——AI回复的开头预填字符串。它的作用是把"时间过渡"融入场景的第一句叙事，让玩家感受到这个特殊场景和主线之间有时间间隔，不会突兀。

不需要严格的天数（"三天后"等），可以是任意叙事性的时间过渡（"那一日"、"几日后"、"又过了几日"等）。

**机制**：每个洛书晴阶段，防线/顺从度的边界刚好等于下一阶段的跳转门槛。

|---------|------------|-------------|

**触发逻辑**：

```text
每轮AI更新洛书晴防线/顺从度
    ↓
脚本截断：不超过当前阶段上限
    ↓
检查：数值是否到达上限 + 云霜凝阶段是否满足同步
    ↓
满足 → 阶段+1，上限放宽 → 下轮注入"刚突破"指引
不满足 → 阶段不动 → 下轮注入"等待师父"指引
未到上限 → 正常 → 下轮注入"正常推进"指引
```

**动态软约束**（每轮根据当前状态注入不同提示）：

```text
情况A（数值未达上限）：
"洛书晴当前阶段X，状态正常，可继续根据剧情推进数值变化"

情况B（数值已达上限 且 云霜凝未同步）：
"洛书晴当前阶段X的心理防线/顺从度已达本阶段极限，
本轮请勿再描写防线下降或顺从度上升，
描写她在某种无形阻力前的犹豫、挣扎或保持当前状态"

情况C（云霜凝同步条件刚满足，阶段刚跳转）：
"洛书晴刚突破到阶段Y，状态指引：可以继续推进，
描写她突破后的细微变化"
```

每轮注入的指引根据当前状态动态切换，AI看到的不是永久指令，而是当前回合的指令——避免AI被"训练"成永久不动。
