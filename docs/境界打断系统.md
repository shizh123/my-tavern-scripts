# 境界打断系统

> 创建日期：2026-01-17
> 文件位置：`src/赵霞/脚本/游戏逻辑/boundaryInterruption.ts`
> 状态：已实现

## 概述

境界打断系统控制丈夫苏文在日常阶段对玩家越界行为的介入。当玩家尝试超出当前境界允许的身体部位互动时，会根据苏文状态和怀疑度触发打断。

---

## Day 5 豁免

**Day 5（结局日）不触发境界打断系统。**

设计理由：

- Day 5 是结局日，玩家已经走到游戏尾声
- 不同结局有不同的剧情走向，不应该被打断系统干扰
- 后日谈阶段根据结局类型决定是否重新激活打断系统

## 打断概率计算

```
打断概率 = 丈夫怀疑度 × 0.5%
```

**示例**：
- 怀疑度 20 → 10% 打断概率
- 怀疑度 60 → 30% 打断概率
- 怀疑度 100 → 50% 打断概率

## 超阶段行为惩罚

当玩家尝试超出当前境界允许的部位互动时：

### 怀疑度惩罚

| 苏文状态 | 基础惩罚 | 最大惩罚 |
|---------|---------|---------|
| 在家可打断 | 5 + 跨境界数×5 | +15 |
| 外出/睡觉 | 10 + 跨境界数×5 | +20 |

**真相模式减半**：已进入过梦境后，惩罚减半

### 严重性判定

| 跨境界数 | 严重性 | 后果 |
|---------|--------|------|
| 1 | 中等 | 赵霞拒绝，增加怀疑度 |
| 2 | 严重 | 苏文可能打断，大幅增加怀疑度 |
| ≥3 | 严重 | 可能触发坏结局（怀疑度达100） |

## 打断模板

### 严重打断模板（5种）

苏文突然出现/回来的场景：

1. **脚步声打断** - 门外传来脚步声
2. **门突然打开** - 苏文推门而入
3. **电话响起** - 苏文来电说马上到
4. **敲门声** - 苏文忘带钥匙敲门
5. **提前回来** - 苏文比预期早归

### 中等打断模板（5种）

赵霞自己的抗拒/犹豫：

1. **本能后退** - 下意识后退
2. **握住手腕** - 阻止进一步动作
3. **转移话题** - 借口离开
4. **身体僵硬** - 愣住不动
5. **突然清醒** - 意识到不对

### 拒绝模板

未触发打断但超阶段行为被拒绝时使用，分为：
- 苏文在家时的拒绝（更紧张）
- 苏文外出时的拒绝（理性克制）

## 核心函数

### `checkBoundaryInterruption(data, userInput)`

检测并处理境界打断。

**返回值**：
```typescript
interface InterruptionCheckResult {
  shouldInterrupt: boolean;     // 是否应该打断
  severity: '无' | '轻微' | '中等' | '严重';
  violatedParts: string[];      // 违规部位列表
  realmGap: number;             // 跨越境界数
  interruptProbability: number; // 打断概率
  wasInterrupted: boolean;      // 是否实际触发打断
  correctionPrompt?: string;    // 修正提示词
  triggerBadEnd: boolean;       // 是否触发坏结局
  penalties?: {
    怀疑度增加?: number;
    混乱度增加?: number;
  };
}
```

### `applyInterruptionResult(data, result)`

应用打断结果到游戏数据：
- 增加怀疑度
- 增加混乱度（完美记忆路线豁免）
- 触发坏结局（如果条件满足）

---

## 梦境豁免

**重要**：梦境中不触发打断系统。

- 检查条件：`data.世界.游戏阶段 === '梦境'`
- 原因：赵霞在睡觉做梦，苏文不可能发现
- 实现：`applyInterruptionResult()` 会检查并跳过

## 与其他系统的交互

1. **丈夫怀疑度系统**：打断检测影响怀疑度增长
2. **外观系统**：境界决定服装/妆容的可见程度
3. **坏结局系统**：怀疑度达100触发发现结局
4. **混乱结局系统**：完美记忆路线可豁免混乱度惩罚
