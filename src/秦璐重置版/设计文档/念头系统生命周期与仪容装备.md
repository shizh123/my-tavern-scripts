> 状态：**草案待审**（2026-07-03 讨论产出，未实现任何代码）
> 议题来源：①仪容/穿搭/妆容/道具应成为玩家主动追求的东西，挂什么数据才有动力；
> ②后期数值堆彻后角色本来就服从，念头植入趋于鸡肋，系统如何不失效。

## 〇、总纲：一句话答案

- 仪容的动力问题 → **挂进念头培育主循环**（加速/越级/风险/阶段外显），不做独立收集系统。
- 念头后期鸡肋问题 → 不给数值续命，让念头系统的价值随进程**三段换挡**：
  **前期卖权限 → 中期卖效率 → 后期卖定制（行为编程）**；
  且**双线并行**保证玩家永远同时处在多个生命周期段（见 §1.4）。
- 特殊剧情 → **奖励性质，不做副本**（云霜凝教训，见 §四）。

核心区分：**服从 ≠ 内化**。数值给的是"你叫她做她照做"；念头给的是"她不用你开口、
自己心痒着做，并真心以为是自己的想法"。前者会毕业，后者是本作幻想内核，永不毕业。
10 大类的最后三档（身份认同/绝对服从/家庭替代）本来就不是"更高权限"，是"她是谁"的改写。

## 一、三段换挡（念头系统生命周期）

| 阶段 | 念头卖什么 | 稀缺资源 | 主要系统 |
|------|-----------|---------|---------|
| 前期（阶段1-2） | **权限**：能不能种进去 | 有效阶段、心防松动窗口 | 越级闸门、判定悬念（现有系统，已服务此段） |
| 中期（阶段3-4） | **效率**：长得快不快 | 楼数、习惯槽周转 | 仪容加速、苏文时机管理、卖习惯再投资 |
| 后期（阶段5） | **定制**：她是谁、每轮怎么演 | 习惯槽×5、念头具体度 | 习惯 build、组合配方、精确植入 |

### 1.1 后期核心一：习惯槽 = build 系统

- 习惯 baseline 每轮驱动 AI 演绎 → **习惯就是内容生成器**，装什么槽她每场戏就怎么演。
- 后期玩法从"推数值"换成"这 5 个槽装什么最色"：装「晨起口侍」还是「外出无内」？
- 换 build = 变卖重种。卖习惯从"腾位的无奈之举"升格为"重新编程"。
- 数值系统会毕业，内容定制不会——它的产出是每一轮正文。

### 1.2 后期核心二：组合配方触发惊喜内容（配方全隐藏）

- 专属场景/结局由**特定习惯组合**触发，不用数值门槛（数值后期无区分度）。
  - 例：「身份认同类习惯×2 + 家庭替代类×1」同时在槽 → 某个隐藏场景降临。
- **配方绝不可见**（§四铁律2）：不做任务清单，条件恰好凑齐时惊喜降临；
  解锁后最多留模糊收藏痕迹（"隐藏场景×1"），不提示下一个的存在。
- 数值到顶后，追求从"更高"换成隐性的"凑出化学反应"。
- 与《潜伏式道具组合》是同一骨架，实现时并轨设计。

### 1.3 后期核心三：念头 = 行为编程

- 前期念头是抽象种子（"想靠在他肩上"），后期念头是**具体行为程序**（"每天早上用口叫醒儿子"）。
- 具体度需要字数 → 字数上限升级做成网店后期商品（见 §三）。

### 1.4 双线并行（已确认：不是串行开档）

秦璐、苏梦**同时推进**，这是三段换挡问题更漂亮的结构解：两条线进度永远不同步，
玩家在任意时刻都同时处在多个生命周期段——每套系统总在某条线上活着，
不需要强行让单条线"永不毕业"。

- **进度差即玩法**：秦璐后期（定制/build）时苏梦还在前中期（权限/效率），
  同一个系统对两条线卖不同的东西。
- **培育槽就是分配决策**（v0.17 修订，取代 v0.15 的"每楼合计 3 条"流量制）：
  每角色 判定中+培育中 ≤ 3 槽（植入扩容特权 → 4），槽满须等成熟/过期或主动退回。
  库存制比流量制更硬：双线的张力从"这轮给谁"变成"有限的心理带宽先开发谁"。
- **耳濡目染（已确认方向：单向污染）**：**只有先堕落的带坏后堕落的**——
  阶段更高的一方已有某类型成熟习惯、且两人同时在场时，另一方同类型念头培育加速。
  单向性自带策略骨架：主攻一个当"污染源"，用她带快另一条线。
  也给了玩家刻意安排两人同场的理由（与在场角色机制直接联动）。
  ⚠ 实现前提：习惯目前不存类型（仅 内容+形成楼层），转习惯时需把类型带上（小 schema 改动）。
- **货币/仓库共享**：一个钱包逼玩家跨线取舍，实现也最简。
- **双人共犯组合**：跨角色的习惯组合配方（如两人各有某习惯 → 触发共犯场景），
  归入 §1.2 的隐藏配方池。
- 诚实的边界依旧成立：双线都走完就该收束到结局，
  **不为"系统永不鸡肋"过度设计**——够撑到玩家自然想看结局就算赢。

## 二、仪容装备系统（四个钩子，全部复用现有数据）

> 守两条老原则：**数值精简铁律**（不加任何新数值）；**奖励感原则**（解锁立见爽点）。

| # | 钩子 | 效果 | 挂的现有数据 | 生命周期 |
|---|------|------|-------------|---------|
| 1 | **培育加速**（主钩子） | 穿戴中，匹配类型倾向的培育中念头每楼 +0.5~1 | 念头开发进度 | 中期主力，后期失效 |
| 2 | **越级钥匙** | 穿戴中，匹配类型有效阶段 +1（不用等心防松动窗口） | getEffectiveStage 的道具 TODO | 前中期，后期自然失效 |
| 3 | **疑心风险对冲** | 高暴露装备在苏文在家时段疑心 +X/楼 | 苏文疑心值、作息黑盒 | **后期免疫贬值**：疑心不是她的数值不会毕业，她越堕落、眼皮下能干的事越极端，张力随堕落度上升 |
| 4 | **阶段门槛/外显** | 她愿不愿意穿由阶段决定；穿着 = 堕落进度的可见证明 | 当前阶段 | 全程，免费的进度棘轮 |

### 2.1 实施要点

- **每件装备 = { 部位, 类型倾向(10大类中若干), 加速值, 风险度, 阶段门槛 }**，装备表脚本侧常量。
- **复用 `系统.道具状态` Record**（未购买/已购买/使用中），"使用中"= 穿戴中，schema 几乎不动。
- **仪容字段归属切一刀**：玩家装备的"件"归脚本管理（AI 不许改，防一轮写没）；
  日常穿着描述仍归 AI（`服装细节` 维持 AI 每轮自由写）。
- **效果全部脚本硬算**，不靠 AI 自觉；快照只加一行"她身上穿着X"级别描述（数据隔离）。
- **首穿事件**：每件装备首次穿上身触发一次性专属演绎注入（"她今天第一次穿上X"），
  性场景立见爽点——对标性癖设计原则，不能只是 UI 换个字符串。
- 建议起步只做钩子 1+4（最简单、动力最直接），2、3 做成高级道具属性后续加。

### 2.2 仪容的后期换挡

加速失效后，装备转型为**演出服**：与习惯联动决定演绎场景/方向（穿什么，习惯往哪个方向演），
加上钩子 3 的背德演出玩法。仪容不与念头系统一起贬值。

## 三、网店定位（对标云霜凝商店，但卖的东西分层）

### 3.1 经济闭环

现状：货币唯一来源 = 卖习惯（100/个，须满5），零去处。网店上线后：

```
种念头 → 成习惯 → 卖习惯换钱 → 买装备/权限 → 加速种更越界的念头
```

习惯槽成为生产资料；定价拉开档次逼玩家在"留习惯当 buff"和"卖习惯换装备"间取舍。

### 3.2 商品分层（按生命周期）

| 层 | 商品 | 服务哪段 |
|----|------|---------|
| 仪容装备 | 加速件（便宜，1-2个习惯的钱）、越级钥匙件（贵，要攒） | 中期 |
| 消耗品 | 借口道具（冻结疑心）、安神药/头孢酒（既定越级道具） | 中后期 |
| **系统权限** | 植入额度 +1/楼（扩容3条上限）、念头字数 10→20（精确编程）、免判定直植（跳过1轮延迟） | **后期主力**，货币终极去处 |
| 演出服 | 不加速，与习惯联动定演绎方向 | 后期 |

> 系统权限商品是后期货币最肥的去处，也让"植入"这个动作本身持续进化——
> 后期玩家买的不是她的东西，是**改造工具本身的升级**。

## 四、特殊剧情设计原则（云霜凝教训：奖励不做副本）

> 病根诊断：云霜凝的特殊剧情（场景引擎 v3c，14 个多轮 beat 强制场景）**可见且必经**，
> 奖励性质的内容变成了玩家只能照设计走的副本。秦璐立三条铁律：

1. **主循环零依赖**：念头→习惯→数值→结局，纯自由玩必须能走通全程。
   特殊剧情只加分，从不设卡——任何进度都不得以"看过某场景"为前置。
2. **配方全隐藏**：触发条件不展示、不做任务清单。条件恰好凑齐时惊喜降临，
   玩家事前不知道这个配方存在。解锁后只留模糊收藏痕迹，不提示下一个。
3. **一次性短注入，不做多轮 beat 强制**：特殊剧情 = 一次 prompt 注入
   （"今晚发生了这样一幕……"），演完立即把控制权还给自由演绎。
   形态对标首穿事件，绝不复刻 v3c 的多轮锁定。
   注入须防 ROLL 重发（MVU 持久字段做幂等门控——老教训）。

惊喜的经济学：稀有才有惊喜感。宁少勿滥，宁短勿长。

## 五、强行植入与三振坏结局（已确认方向）

> 前提：越级闸门 v0.14 起为测试临时关闭，**正式版恢复**。本节是恢复闸门时的配套设计，
> 让闸门从"拦一下"升级为"拦 + 诱惑 + 三振坏结局"一条龙。
> 解决的软肋：现在退回零代价，玩家可以无脑刷高阶念头赌通过——三振给退回装上牙齿。

### 5.1 镜像原则：奖励藏、惩罚亮

与 §四"配方全隐藏"是**镜像关系**，方向必须相反：

- 隐藏的奖励 = 惊喜（爽）；隐藏的惩罚 = 背刺（玩家觉得被耍）。
- 三振计数**绝不静默**，升级预警让玩家清楚看到刀架在脖子上。
  玩家走到坏结局应是"我知道我在玩火，自找的"，不是"什么鬼"。

### 5.2 流程

1. 植入高阶念头 → 判定超出有效阶段 → UI 提示：
   **"当前角色阶段无法植入该念头，是否强行植入？"**
2. 选择强行 → 念头下一楼仍被退回（不进培育），但：
   - **即时代价 + 微小收益（诱惑派，已确认）**：具体项待平衡，方向如
     苏文疑心↑（她被吓到的异样瞒不过丈夫）+ 目标依存/情绪被强行撬动一丝——
     "疼但有点上头"的赌博感，让理性玩家也会心痒。
   - 该角色的**强行计数 +1**。
3. 升级预警（惩罚亮）：
   - 第 1 次：她惊恐退缩——"她还接受不了，你把她吓到了"
   - 第 2 次：警告拉满——"她开始怕你了，再逼一次可能出事"（UI 标红 + 快照让 AI 演出戒备）
   - 第 3 次：**坏结局，锁死存档**
4. **清零规则（已确认：连续计数）**：该角色任一念头正常培育成功即清零。
   只有"连着逼三次"才触发，不翻旧账。

### 5.3 范围与结局形态（已确认）

- 计数**按角色分**（谁被逼谁记账，秦璐崩溃不连累苏梦的计数）。
- 但任一角色三振 → **全局坏结局、锁死存档**——她崩溃了，这个家就完了
  （叙事自洽：崩溃意味着告发/决裂，游戏整体终结）。
- 与心防松动窗口互补：窗口是合法越级通道（安全，等时机），强行是危险捷径（赌命）。
  想越级永远有两条路，玩家自选风险偏好。

### 5.4 带出的新 scope

- 这是游戏**第一个坏结局**，需要顺带起一套坏结局呈现层（对标云霜凝终局覆盖层：
  覆盖 UI + 脚本停止正常循环）。"锁死"是呈现层语义——酒馆里玩家仍可删楼回档，不强防。
- 强行计数用 MVU 持久字段（防 reload 重置——模块变量老教训）。

## 六、苏文定位（已确认：没有终局，永远被瞒在鼓里的幸福丈夫）

> 苏文没有终局弧线。他的"结局"就是从头到尾被瞒得很好，做一个幸福的丈夫。

设计含义：

- **不做**绿帽奴化/摊牌/离婚/移除路线（明确区别于云霜凝苗广的屈辱→默许弧线——
  苗广被收编后风险感即死；苏文永不收束，"瞒"全程保鲜）。
- **"瞒住"是玩家的成就品，不是保底**：他越幸福，戏剧反讽越浓
  （他在餐桌上夸妻子最近气色真好……）。反讽场景是隐藏配方池的天然素材。
- **疑心线全程有效** → §二 钩子3（仪容-疑心风险）全生命周期有效，由推断升级为定论。
- **苏文 = 反向养成对象**：玩家养成两条女性线的同时，"养成"苏文的幸福蒙昧。
  加速房/作息黑盒机制全程保留。
- **推论（待确认）**：疑心爆表应构成**第二个坏结局**（被发现 = 家庭崩塌 = 锁档），
  与三振坏结局并列——两个坏结局分别惩罚"逼太狠"和"瞒太糙"，正好覆盖两种玩家失误。
  借口/冻结机制就是这条线的操作空间。
