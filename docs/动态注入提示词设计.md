# 赵霞项目 - 动态注入提示词设计

## 概述

本文档展示在游戏运行过程中，脚本动态注入给 AI 的提示词。
这些提示词**根据当前游戏状态实时生成**，不同状态下内容完全不同。

## 核心设计原则

### ⚠️ 约束 vs 限制

**错误方式：告诉AI"只能写什么"**
```
✅ 午后阳光、慵懒氛围
❌ 清晨晨光、深夜月光
```
问题：AI被限制住，每次都写"午后阳光"，输出模板化、同质化。

**正确方式：告诉AI"不要犯什么错"**
```
⚠️ 当前是下午，避免描写深夜/清晨的光线
```
好处：AI知道边界在哪，但有充分的创作自由（可以是晴天、阴天、下雨等）。

### 约束原则

1. **只约束容易出错的点** - 不要事无巨细地规定
2. **负向约束优于正向限制** - "不要写X"比"只能写Y"更好
3. **保留创作空间** - AI应该能自由发挥场景细节
4. **状态提醒而非模板** - 告诉AI当前状态，让它自己判断如何描写

## 动态变化维度

| 维度 | 可能的值 | 约束重点 |
|------|---------|---------|
| 游戏阶段 | 日常/梦境/结局 | 完全不同的规则集 |
| 时间 | 00:00-23:59 | 避免时间错乱（下午写成深夜） |
| 苏文状态 | 清醒在家/睡觉/外出 | 避免苏文行为与状态矛盾 |
| 梦境场景 | 场景1~5 | 赵霞年龄、身份、禁止称呼 |
| 记忆连续性 | 0~3级 | 赵霞对玩家的认知程度 |
| 境界 | 1~5 | 赵霞对亲密行为的反应 |
| 游戏模式 | 纯爱/真相 | 部位进度更新规则 |

### 场景1：日常（苏文清醒在家）

```

# 当前状态
境界：1/5（无感）| 模式：纯爱
苏文：客厅（清醒）

# 约束提醒
⚠️ 当前是下午，避免描写深夜/清晨的光线场景
⚠️ 苏文在家清醒，不要让他突然消失、睡着或外出
⚠️ 境界1，赵霞对超出母子关系的亲密行为会困惑或回避
```

## 动态变化对比

### 同一时间（23:00），苏文状态不同

| 苏文状态 | 约束提醒 |
|---------|---------|
| 睡觉 | `⚠️ 苏文已入睡，不要让他说话、走动或离开卧室` |
| 失眠 | `⚠️ 苏文失眠中，可能随时走动，注意被发现的风险` |
| 外出 | `⚠️ 苏文不在家，不要让他突然出现` |

### 同一场景（梦境），记忆连续性不同

| 连续性 | 状态描述（供AI参考） |
|--------|---------|
| 0级 | 赵霞不认识玩家，是第一次见面的陌生人 |
| 1级 | 赵霞有模糊印象，觉得玩家有些眼熟 |
| 2级 | 赵霞认出玩家，对他有特殊的好感 |
| 3级 | 赵霞视玩家为命中注定之人，多年来一直在等他 |

### 同一阶段（日常），境界不同

| 境界 | 约束提醒 |
|------|---------|
| 1 | `⚠️ 赵霞对超出母子关系的行为会困惑或回避` |
| 3 | `⚠️ 赵霞对亲密行为会犹豫但不完全抗拒` |
| 5 | `⚠️ 赵霞已完全沦陷，几乎不会拒绝任何要求` |

## 实现位置

这些动态提示词在 `promptInjection.ts` 中生成：

### 核心函数

| 函数 | 作用 |
|------|------|
| `generatePhaseAwareStatePrompt()` | 阶段感知状态（自动切换日常/梦境） |
| `generateDailyPhasePrompt()` | 日常阶段提示 + 约束提醒 |
| `generateDreamPhasePrompt()` | 梦境阶段提示 + 约束提醒 |
| `generateBodyPartInfluencePrompt()` | 部位开发影响 |

### 约束提醒函数（2026-01-16 新增）

| 函数 | 作用 |
|------|------|
| `getTimeConstraintReminder()` | 时间约束提醒（避免时间错乱） |
| `getHusbandConstraintReminder()` | 苏文状态约束提醒（避免行为矛盾） |
| `getRealmConstraintReminder()` | 境界约束提醒（赵霞反应倾向） |
| `getDreamConstraintReminder()` | 梦境约束提醒（完整输出） |

根据当前 `stat_data` 的值，实时构建不同的提示词注入给 AI。

### 2026-01-16 梦境场景记忆隔离

**问题描述**：梦境场景之间的服装/外貌被错误继承

当玩家进入新的梦境场景（如场景2：17岁）时，AI会参考对话历史中上一个场景（场景1：16岁）的服装描写，导致服装被错误继承。

**根本原因**：
1. 记忆连续性系统只应继承"心理记忆"（对玩家的感情、认识程度）
2. 但AI会从对话历史中看到之前场景的服装描写，然后延续这些描写
3. 每个梦境场景应该是独立的时间片段，服装/外貌应该根据当前场景年龄重新描写

**解决方案**：在两个位置添加明确的隔离约束

**1. 修改 `generateEnhancedMemoryContinuityPrompt()`** (dreamKeywordDetection.ts)

在记忆连续性提示词中添加【记忆隔离】区块：
```
【⚠️ 记忆隔离 - 非常重要】
记忆连续性只继承**心理记忆**，不继承**物理状态**：
✅ 应该继承：对玩家的感情、认识程度、情感变化、互动经历
❌ 不应继承：服装、外貌、妆容、年龄外表、场景环境

当前场景是XX岁的赵霞：
- 她的外貌、穿着必须符合XX岁的设定
- 完全忽略对话历史中其他场景的服装描写
- 每个梦境场景的服装由AI根据当前场景年龄自由创作
```

**2. 修改各场景入口提示词** (promptInjection.ts)

在【禁止】列表中添加：
```
- 不要继承其他梦境场景的服装/外貌描写，必须根据XX岁设定重新描写
```

**修改的函数**：
- `generateEnhancedMemoryContinuityPrompt()` - 添加【记忆隔离】区块
- `generateScene1EntryReplacement()` - 添加服装隔离禁止项
- `generateScene2EntryReplacement()` - 添加服装隔离禁止项
- `generateScene3EntryReplacement()` - 添加服装隔离禁止项
- `generateScene4EntryReplacement()` - 添加服装隔离禁止项
- `generateDreamEntryReplacement()` - 添加服装隔离禁止项（通用入口）

**设计原则**：
- 记忆连续性 ≠ 状态继承
- 心理记忆跨场景保持，物理状态每场景独立
- 告诉AI"什么不应该继承"而非"应该继承什么"

**问题描述**：与秦璐项目对比，赵霞的 `变量更新规则.yaml` 缺少状态一致性规则和约束速查表

| 部位开发影响参考 | ❌ 无 | ✅ 已添加 |

**补充内容**：

   - 苏文状态约束表
   - 游戏阶段约束表
   - 禁止描写清单

### 2026-01-16 梦境约束统一化

**问题描述**：场景5的提示词缺少"不要叫儿子"等核心约束

**解决方案**：创建 `getDreamConstraintReminder()` 函数，统一管理梦境约束

**重要说明**：
这些动态注入的提示词**不会保存到聊天历史中**，AI 每次请求都是第一次看到这些约束。
因此必须每次都输出完整的约束列表，不能使用轮换机制。

**约束列表**（每次都完整输出）：
1. 不要让赵霞叫玩家"儿子"或任何亲属称呼
2. 赵霞不知道自己在梦中，不要打破第四面墙
3. 赵霞是X岁，不要让她拥有超出X岁的知识
4. 梦境赵霞不知道现实中的日常对话内容
5. 不输出 `<HusbandThought>`，不更新 `/现实数据/*`

**场景特殊约束**（对应场景显示）：
- 场景2：苏文迟到，不要让他登场
- 场景4：刚大吵一架，苏文不出场

**修改函数**：
- `getDreamConstraintReminder(age, sceneNum)` - 新增函数
- `generateDreamPhasePrompt()` - 使用新函数替代硬编码约束
- `generateScene5EntryReplacement()` / `generateScene5StepPrompt()` / `generateFreePlayPrompt()` (scene5System.ts) - 添加缺失的核心约束
