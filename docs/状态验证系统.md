# 状态验证系统

> 创建日期：2026-01-17
> 文件位置：`src/赵霞/脚本/游戏逻辑/stateValidation.ts`
> 状态：已实现

## 概述

状态验证系统负责确保所有游戏数值在合理范围内，自动修正异常数据，并根据依存度自动更新境界。

## 核心功能

1. **数值范围检查**：确保所有数值在0-100范围内
2. **境界自动更新**：根据依存度自动计算境界
3. **部位进度限制**：场景限制和模式限制
4. **威胁数值监控**：检测坏结局触发条件
5. **时间范围检查**：天数1-5，小时0-23

---

## 核心函数

### `validateAndFixState(data)`

验证并修正所有数值。

**检查项目**：

1. **核心数值**：
   - 依存度：0-100
   - 道德底线：0-100
   - 对丈夫依存度：-50 ~ 100

2. **境界自动更新**：
   ```typescript
   function calculateRealm(依存度: number): number {
     if (依存度 < 20) return 1;
     if (依存度 < 40) return 2;
     if (依存度 < 60) return 3;
     if (依存度 < 80) return 4;
     return 5;
   }
   ```

3. **部位进度**：
   - 范围：0-100
   - 纯爱模式下精神进度强制锁定为0

4. **威胁数值**：
   - 记忆混乱度：0-100
   - 丈夫怀疑度：0-100

5. **时间范围**：
   - 天数：1-5
   - 小时：0-23

**返回值**：
```typescript
{
  fixed: boolean;      // 是否有修正
  changes: string[];   // 修正记录列表
}
```

---

### `checkRealmChange(data)`

检测境界变化。

**用途**：在境界变化时触发相关事件（如广播、日志）

**返回值**：
```typescript
{
  changed: boolean;
  oldRealm: number;
  newRealm: number;
}
```

---

### 境界名称函数

**纯爱模式**：`getRealmNamePure(realm)`
- 1: 初识
- 2: 好友
- 3: 暧昧
- 4: 热恋
- 5: 真爱

**真相模式**：`getRealmNameTruth(realm)`
- 1: 初染
- 2: 迷途
- 3: 溺深
- 4: 归虚
- 5: 焚誓

---

## 特殊规则

### 精神进度场景限制

- **纯爱模式**：精神进度强制为0
- **真相模式场景1-4**：只能开发身体部位（嘴巴/胸部/下体/后穴）
- **真相模式场景5**：只能开发精神

### 坏结局触发检测

**记忆混乱度 ≥ 100**：
- 触发精神崩溃结局（混乱结局）
- 无豁免

**丈夫怀疑度 ≥ 100**：
- 触发发现结局
- **梦境豁免**：梦境中怀疑度达到100不触发（赵霞在睡觉，苏文无法发现）

---

## 与其他系统的交互

1. **游戏逻辑主入口**：每轮调用 `validateAndFixState()`
2. **外观系统**：境界变化后更新服装约束
3. **结局系统**：威胁数值超限触发坏结局
