# 秦璐重置版 — 苏文系统（刺激源 + 越级惩戒 + 拟人化作息）

> 状态：定位与核心机制已敲定，作息表/数值待细化。
> 记录时间：2026-06-24
> 关联：《去时间化总原则.md》《心防松动窗口.md》《商店道具系统.md》

---

## 〇、设计初衷（最高纲领）

**苏文的本质是「偷偷摸摸进行的刺激感」的来源，不是惩罚玩家的警察。**

- 旧版误区：苏文是"惩罚系统"（危险累计→疑心→突脸），制造的是**焦虑（怕被抓）**，导致玩家想消除苏文、苏文一出现就烦。
- 重置版定位：苏文是**刺激催化剂**。他在场、且在女角色身边（同房/隔壁），"在他眼皮底下进行"的偷腥快感越强 → 机制上**加速念头成熟**。**玩家应该想要苏文在场**——他不在，就不刺激、也没加速了。
- 一句话：**刺激来自"风险悬念"，不来自"惩罚代价"。**

## 一、苏文的三重职责

| 职责 | 说明 |
|------|------|
| ① 刺激源（核心） | 苏文在特定房间且女角色在其附近 → **加速念头成熟**（在他眼皮底下偷偷调教，进展更快）。偷腥快感的机制兑现。 |
| ② 越级惩戒闸（关键） | 玩家在**低阶段硬做高阶段行为**时，苏文打断来惩戒。这是"循序渐进"的守门人，**这才是打断系统的真正功能**。 |
| ③ 疑心值的承载体 | 疑心值保留（游戏特点之一），玩家可用道具降低（详见《商店道具系统.md》，数值待设计）。 |

> 注意：「被发现」**理论上不可能**——玩家可删消息重生成。所以苏文不是"会抓到你"的真实威胁。它的威慑力体现在 ② 越级惩戒，而非 ① 的"被抓"。

## 二、刺激加成系统（职责①）：苏文在场 → 加速念头成熟

> 核心简化（已确认·最终）：加速**只看苏文位置**。苏文待在特定房间 → 念头加速成熟。**女角色在哪、是否同房，完全不参与判定**。脚本只读苏文一个变量，极简。

### 2.1 加速触发条件（已确认·最终）

**`苏文位置 ∈ {餐厅, 客厅, 主卧}` → 该楼念头加速成熟。**

- **加速房间**（已确认）：**餐厅、客厅、主卧**。
- **女角色位置不参与**：秦璐在厨房、苏梦在自己房间、甚至和苏文同在客厅——都照样加速。
  - 理由（已确认）：房间够大（如客厅有前后沙发），"同空间也能偷偷来"，照样刺激照样加成。不需要算距离、相邻、同空间，**不需要房间邻接表**。
- 苏文**不在家**（外出工作）或在**非加速房间**（厨房/浴室/各自房间）→ 不加速，正常速度。

### 2.2 地点枚举（补齐三人房间，已确认）

重置版地点枚举（schema `Location`）：

| 地点 | 是否加速房间 |
|------|------|
| 餐厅 | ✅ |
| 客厅 | ✅ |
| 主卧 | ✅ |
| 厨房 | ❌ |
| 浴室 | ❌ |
| 秦璐房间（新增） | ❌ |
| 苏梦房间（新增） | ❌ |
| 主角房间（旧有，保留） | ❌ |
| 外面（外出工作=安全期） | ❌ |

- 新增**秦璐房间、苏梦房间**，连同旧有**主角房间**，三人各有独立房间。
- 苏文位置 → **脚本**填（楼层游标，可靠）。女角色位置 → **AI 选择题**（从上表枚举选）+ 脚本兜底默认（女角色位置不影响加速，但其它叙事/显示可能用到）。

### 2.3 加速效果（已确认方向，强度待定）

- 满足条件的那一楼，念头培育进度**加速**，使成熟所需推进**快约 1/3**（等效：那些楼进度 ×1.5，或所需楼数打 2/3 折）。
- **动态生效**（已确认）：**只算加速条件成立的那些楼**；条件不成立的楼按正常速度。不是植入时一次性定死。
- 实现极简：每楼判断 `苏文位置 ∈ {餐厅,客厅,主卧}` → 是 → 这一楼多加进度。**比对的是脚本写好的苏文位置变量，不是 AI 文本关键词**，可靠。

### 2.4 数值不变，只是更快（已确认）

- 加速**只缩短成熟所需楼数**，不改成熟时的数值奖励大小（依存/侵蚀变化仍只在成熟那一刻按念头难度结算，见《念头培育与时间系统.md》）。

### 2.5 显示（已确认）

- 处于加速状态时，用**发光字体 / 流动特效**标示（如念头进度条发光、苏文位置高亮），**无需任何文字说明**。

> 待商量：加速强度 1/3 是否合适——需连同"念头等级/难度体系"一起看（见§七待办）。

## 三、越级惩戒：打断系统的真正功能（职责②）

### 3.1 触发条件（重定义）

- 打断**不再**由"危险时间累积到顶"触发。
- 打断 = **玩家在低阶段硬做高阶段行为**时触发（越级惩戒）。
  - 例：阶段1（抵抗期）玩家就想直接上手亲密接触/性行为 → 苏文打断："你妈才跟你聊两句，苏文就推门进来了。"
  - 与念头判定/阶段闸门联动：玩家植入或推进**远超当前阶段**的行为 → 触发惩戒打断。

### 3.2 与心防松动窗口/道具的关系

- **心防松动窗口期 / 安眠药 / 住院期间**：越级被合法解锁（见《心防松动窗口.md》《商店道具系统.md》），此时**不触发越级惩戒打断**——因为越级是被允许的。
- 即：越级惩戒只惩罚"**没有解锁资格还硬来**"的越级，不惩罚"合法越级"。

### 3.3 打断的形式（弱化、不真惩罚玩家）

- 打断是**剧情演出**（苏文出现、行为被迫中止），不是删档式惩罚。
- 偶尔的"虚惊"（差点撞见、擦肩）可作为**制造心跳的调味**，频率要低，不打断正常游戏节奏（修复旧版"发一句就突脸"）。

### 3.4 借口机制：保留，但重新设计（已确认）

旧版打断后的"苏文质问 → 角色亲口找借口 → AI 判定成功/失败 → 疑心冻结/惩罚"这套交互**值得保留**（玩家很认可），但要重做。

**旧版借口机制现状（供重设计参照）**：
- 触发：打断事件 `待触发=true` → AI 演质问 → `等待借口=true` → 玩家下一轮让角色说借口 → AI 判 `借口结果`。
- 自动失败条件：玩家替角色说借口 / 没说明确时间限额 / 时间 > 4 小时。
- 正常判定：疑心值高低 + 衣着/气氛 + 借口合理性。
- 成功 → 疑心冻结 2 小时，角色须按承诺时间离开，超时疑心 +8。
- 失败 → 疑心额外 +10（总 +20），角色必须离开主角房间。

**重设计待办（细则后议）**：
- [ ] 触发场景从"停留超时"改为"越级惩戒"语境后，借口话术/判定标准如何调整。
- [ ] "离开期限/超时惩罚"是否保留——本系统已废除巡逻与停留累积，离开期限的意义需重新定义（倾向：简化或去掉时间限额，改楼层或直接去掉）。
- [ ] 疑心冻结改为楼层化（冻结 N 楼，见《去时间化总原则.md》§六）。
- [ ] 借口判定继续走"角色亲口说、玩家不能替说"的核心规则（这条保留）。

### 3.5 巡逻系统：完全废除（已确认）

- 旧版"概率巡逻 + 时间段系数 + 离开期限 + 冷却 + ROLL 可刷"**整套删除**。
- 苏文是否出现在玩家附近，完全由**拟人化作息表**决定（见§四），不再有独立的随机巡逻系统。
- 若作息让苏文路过主角房间门口，那只是作息的自然结果（可作一次"虚惊"调味，见§3.3），不是巡逻机制。

## 四、苏文位置：楼层驱动的黑盒生活节律（已确认）

### 4.1 核心原则

- **楼层 = 时间流逝**：玩家每推进一楼，苏文的生活在继续，位置随楼层变化——就像时间在走、苏文在自然地生活。
- **黑盒、不告知**：楼层 → 位置的映射**原理是脚本内部的事**，不告诉玩家、不告诉 AI。对外只暴露"苏文当前位置/状态"这个**结果**。
  - AI 只负责"演绎苏文在那个位置做什么"，不需要、也不应该知道他为什么在那、下一步去哪。
  - 玩家只看到苏文当前在哪，看不到背后的楼层规律。
- **彻底去时间化**：苏文位置不再依赖任何真实时间/时段。这使去时间化**不再有任何例外**——时间完全退出所有计算。

### 4.2 机制本质：按楼层推进的循环模板游标

苏文位置 = 一个**作息模板**，随楼层推进一段段往下走，走完自动循环到下一天模板。脚本只维护一个"当前游标"（走到模板第几段），对外只输出"苏文此刻在哪"。

- **每段至少停留 2 楼**（已确认：1 楼太短）。各段停留楼数可不同（睡眠/客厅久，吃饭短），模拟真人节奏。
- **不是随机**：模板固定，同一进程可复现（防 ROLL 刷）。
- **一天多少楼无所谓**（已确认）：玩家慢慢聊游标就慢慢走，没有"一天必须几楼"的硬约束。模板循环跑即可。

### 4.3 时间/日期完全交给 AI，与苏文位置彻底解耦（已确认·关键）

- **脚本不更新日期/时间**。日期、时间显示**全部交给 AI** 按剧情自由写，**写不准也没关系**。
- 苏文位置（脚本模板游标）和 日期时间显示（AI 自由写）是**两套独立系统**，不要求对齐——这样才不会出现"脚本算的时间对不上玩家节奏"的违和。
- 脚本只管一件最简单的事：**苏文此刻在哪**。

### 4.4 关键词跳转 = 模板重置（已确认）

- 系统**检测玩家输入的时间跳转关键词**（如"第二天""第二日""几日后的早晨""下午""晚上"等），据此**把苏文模板游标重置/跳到对应作息段**。
  - 例："第二天早上" → 游标跳到次日模板的早晨段（苏文起床/早饭）。
  - 例："几日后" → 游标跳到对应日类型的开头段。
- 跳转**只挪苏文位置游标**：
  - **不更新日期/时间**（那是 AI 的事）。
  - **不推进念头培育进度**（念头只靠正常对话楼层推进，跳转不白嫖、不白嫌）。
- 这复活了旧版 `detectTimeJump` 的关键词检测，但用途改为"重置苏文作息游标"，不再换算时间/影响培育。

### 4.5 作息模板定稿（建议值，每段≥2楼，可调）

**工作日模板（有班）**

| 模板段 | 状态/位置 | 停留楼数 |
|--------|-----------|---------|
| 1 | 在家·主卧→餐厅（起床早饭） | 2 |
| 2 | **外出工作（不在家）** | 7 |
| 3 | 在家·餐厅（晚饭） | 2 |
| 4 | 在家·客厅（看电视） | 3 |
| 5 | 睡眠·主卧 | 2 |

**在家日模板（周末 / 半天班在家段）**

| 模板段 | 状态/位置 | 停留楼数 |
|--------|-----------|---------|
| 1 | 睡眠/赖床·主卧 | 2 |
| 2 | 在家·餐厅（早午饭） | 2 |
| 3 | 在家·客厅 | 3 |
| 4 | 在家·厨房/主卧（走动） | 2 |
| 5 | 在家·客厅 | 3 |
| 6 | 睡眠·主卧 | 3 |

**一周组合（星期由模板序列决定，非真实时间）**

| 星期 | 用哪个模板 | 特例 |
|------|-----------|------|
| 周一/周二/周四 | 工作日模板 | — |
| 周三 | 工作日模板，但**下午段起改在家** | 上午外出→下午在家（意外刺激窗口） |
| 周五 | **上午在家**→下午工作日模板 | 上午反常在家 |
| 周六/周日 | 在家日模板 | — |

> 注：这里的"星期/上午下午"是**模板编排概念**，由楼层游标推进决定，不依赖真实时间。AI 显示的日期可以和它不一致，无妨。

### 4.6 安全期 vs 刺激窗口

| 时段（模板段） | 安全度 |
|------|--------|
| 工作日外出段、周三上午/周五下午外出段 | **完全安全，不触发任何打断** |
| 所有睡眠段 | 安全（睡眠态） |
| **周三下午、周五上午在家** | 意外刺激窗口 |
| 周末整天、工作日早晚在家 | 常规在场，按距离分刺激度 |

### 4.7 玩家可见性（已确认：只知当前位置）

- 状态栏只显示**苏文当前位置/状态**（如"苏文：在家·客厅"、"苏文：外出工作"）。
- **不显示**苏文下一步去哪、还待多久、背后游标规律 → 保留不确定性与刺激。

## 六、与旧版的对照

| 维度 | 旧版（惩罚型） | 重置版（刺激型） |
|------|--------------|----------------|
| 苏文出现 | 危险倒计时/惩罚 | 刺激加成（苏文在餐厅/客厅/主卧 → 念头加速成熟，只看苏文位置） |
| 打断触发 | 危险时间累积到顶（高频突脸） | 仅"低阶段硬做高阶段"越级惩戒（低频） |
| 巡逻系统 | 概率巡逻+时间段系数+离开期限+可ROLL刷 | **完全废除** |
| 借口机制 | 质问→角色找借口→判定→冻结/惩罚 | **保留但重新设计**（核心交互值得留） |
| 玩家诉求 | 想消除苏文 | 想利用苏文制造心跳 |
| 苏文位置 | 时间驱动作息（旧版已有，但服务于威胁） | **楼层驱动黑盒节律**（不告知玩家/AI 原理），位置只显示当前 |
| 疑心值 | 倒计时武器 | 保留为特点，道具可降 |
