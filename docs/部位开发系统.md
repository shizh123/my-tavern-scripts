# 部位开发系统

> 创建日期：2026-01-17
> 文件位置：`src/赵霞/脚本/游戏逻辑/dreamKeywordDetection.ts`
> 状态：已实现

## 概述

部位开发系统负责在梦境中检测玩家输入的关键词，根据匹配结果增加对应部位的开发进度。

## 核心设计

### 五个部位
- **嘴巴**：场景1-4可开发
- **胸部**：场景1-4可开发
- **下体**：场景1-4可开发
- **后穴**：场景1-4可开发
- **精神**：仅场景5可开发

### 数值范围
- 每个部位：0-100%
- 每轮增量上限：5%
- 每晚增量上限：20%

## 核心函数

### `detectBodyPartProgress(text)`

检测文本中的部位关键词。

**返回值**：
```typescript
{
  嘴巴: number;  // 匹配到则为5，否则为0
  胸部: number;
  下体: number;
  后穴: number;
  精神: number;
}
```

---

### `updateBodyPartProgress(data, progress)`

更新部位开发进度。

**规则**：
1. 场景1-4：只更新身体部位，精神强制为0
2. 场景5：只更新精神，身体部位不变
3. 应用每轮/每晚上限

---

### `getBodyPartSummary(data)`

获取部位开发摘要。

**返回值**：
```typescript
{
  嘴巴: { 进度: number; 等级: string; };
  胸部: { 进度: number; 等级: string; };
  下体: { 进度: number; 等级: string; };
  后穴: { 进度: number; 等级: string; };
  精神: { 进度: number; 等级: string; };
  平均进度: number;
  最高部位: string;
}
```

## 进度等级

| 进度范围 | 等级 | 接受程度 |
|---------|------|---------|
| 0-19% | 未开发 | 完全抗拒 |
| 20-39% | 初步开发 | 轻微接受 |
| 40-59% | 明显开发 | 明显接受 |
| 60-79% | 深度开发 | 深度接受 |
| 80-100% | 完全开发 | 完全接受 |

---

## 每晚上限机制

**数据结构**：
```typescript
当晚进度记录: {
  天数: number;
  嘴巴: number;
  胸部: number;
  下体: number;
  后穴: number;
  精神: number;
}
```

**规则**：
- 每个部位每晚最多增加20%
- 跨天时自动重置记录
- 超出上限的增量被忽略

|------|-----------|

| 场景2 | 嘴巴/胸部/下体/后穴 |
| 场景3 | 嘴巴/胸部/下体/后穴 |
| 场景4 | 嘴巴/胸部/下体/后穴 |
| 场景5 | 精神（通过步骤系统控制） |

---

## 与真相模式数值的联动

### 依存度
```
依存度 = (嘴巴 + 胸部 + 下体 + 后穴 + 精神) / 5
```

### 道德底线
```
道德底线 = 100 - 依存度
```

### 境界
```
境界 = Math.ceil(依存度 / 20) 
```

---

## 平衡性设计（2026-01-16）

**设计目标**：4天晚上把全部数值做到80-100%

**检测规则**：
- 只检测玩家输入，不检测 AI 回复
- 避免 AI 描写词汇导致被动增长
- 场景5不受关键词系统控制

---

## 与其他系统的交互

1. **游戏逻辑主入口**：梦境中调用检测和更新
2. **外观系统**：依存度影响境界，境界影响服装
3. **结局系统**：部位完成度影响结局判定
