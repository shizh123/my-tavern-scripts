## 一、状态一致化规则

### 文件位置
[src/赵霞/脚本/游戏逻辑/stateValidation.ts](src/赵霞/脚本/游戏逻辑/stateValidation.ts)

### 功能

#### 1. 数值范围检查

自动限制所有数值在合理范围内：

```typescript
// 核心数值
依存度: 0-100
道德底线: 0-100
对丈夫依存度: -50-100

// 部位进度
嘴巴、胸部、下体、后穴、精神: 0-100

// 威胁数值
记忆混乱度: 0-100
丈夫怀疑度: 0-100

// 时间
当前天数: 1-5
当前小时: 0-23
```

#### 2. 境界自动更新

**计算规则**：
```typescript
依存度 0-19   → 境界1
依存度 20-39  → 境界2
依存度 40-59  → 境界3
依存度 60-79  → 境界4
依存度 80-100 → 境界5
```

**双重名称**：
- 纯爱模式：初识、好友、暧昧、热恋、真爱
- 真相模式：初染、迷途、溺深、归虚、焚誓

#### 3. 境界变化检测

```typescript
checkRealmChange(data);
// 返回: { changed: boolean, oldRealm: number, newRealm: number }
```

**功能**：
- 检测境界是否发生变化
- 输出详细日志（包含双重名称）
- 用于触发境界提升事件

### 核心函数

```typescript
// 验证并修正所有数值
validateAndFixState(data);
// 返回: { fixed: boolean, changes: string[] }

// 计算正确境界
calculateRealm(依存度);
// 返回: 1-5

// 检测境界变化
checkRealmChange(data);
// 返回: { changed, oldRealm, newRealm }

// 获取境界名称
getRealmNamePure(realm);    // 纯爱模式名称
getRealmNameTruth(realm);   // 真相模式名称
```

### 使用示例

```typescript
// 在主游戏循环中
const 验证结果 = validateAndFixState(data);
if (验证结果.fixed) {
  console.info(`状态已自动修正，共 ${验证结果.changes.length} 项变更`);
  验证结果.changes.forEach(change => console.info(`  - ${change}`));
}

const 境界变化 = checkRealmChange(data);
if (境界变化.changed) {
  console.info(`境界提升: ${境界变化.oldRealm} → ${境界变化.newRealm}`);
  // 触发境界提升事件
}
```

## 七、真相模式数值系统（核心设计）

### 设计原理

**纯爱模式** 和 **真相模式** 是**互斥**的两条游戏路线：
- 玩家未进入梦境 → 纯爱模式（伪装成正常恋爱游戏）
- 玩家首次进入梦境 → 真相模式激活，纯爱模式**永久消失**

### 核心公式

真相模式下，所有数值由**部位开发程度**自动计算：

```typescript
// 文件: appearanceSystem.ts

// 依存度 = 所有部位进度的平均值
依存度 = (嘴巴 + 胸部 + 下体 + 后穴 + 精神) / 5

// 道德底线 = 100 - 依存度
道德底线 = 100 - 依存度

// 境界根据依存度自动计算
依存度 0-19   → 境界1（初染）
依存度 20-39  → 境界2（迷途）
依存度 40-59  → 境界3（溺深）
依存度 60-79  → 境界4（归虚）
依存度 80-100 → 境界5（焚誓）
```

### 代码实现

```typescript
// 计算依存度（部位进度平均值）
export function calculateDependencyFromBodyProgress(部位进度: Record<BodyPartName, number>): number {
  const values = ALL_BODY_PARTS.map((part) => 部位进度[part] || 0);
  const average = values.reduce((sum, v) => sum + v, 0) / values.length;
  return Math.round(average);
}

// 计算道德底线
export function calculateMoralBaselineFromDependency(依存度: number): number {
  return Math.max(0, Math.min(100, 100 - 依存度));
}

// 计算境界
export function calculateRealmFromDependency(依存度: number): number {
  if (依存度 >= 80) return 5;
  if (依存度 >= 60) return 4;
  if (依存度 >= 40) return 3;
  if (依存度 >= 20) return 2;
  return 1;
}

// 完整数值更新（在主游戏循环中调用）
export function updateTruthModeValues(data): void {
  if (!data.世界.已进入过梦境) return;  // 只在真相模式执行

  const 新依存度 = calculateDependencyFromBodyProgress(data.赵霞状态.部位进度);
  const 新道德底线 = calculateMoralBaselineFromDependency(新依存度);
  const 新境界 = calculateRealmFromDependency(新依存度);

  data.赵霞状态.依存度 = 新依存度;
  data.赵霞状态.道德底线 = 新道德底线;
  data.赵霞状态.当前境界 = 新境界;
}
```

### 两种模式对比

| 维度 | 纯爱模式 | 真相模式 |
|------|----------|----------|
| **数值控制** | AI生成（好感度等） | 脚本计算（部位开发决定） |
| **境界限制** | 有（限制部位互动） | 无（所有部位同时可开发） |
| **显示文本** | 初识/好友/暧昧/热恋/真爱 | 初染/迷途/溺深/归虚/焚誓 |
| **梦境影响** | 不存在 | 梦境开发直接影响现实数值 |

## 十一、核心数据保护系统

### 文件位置

### 设计目标

防止AI随意修改关键数值，确保游戏数据的一致性和可控性：
1. **部位进度**：只能通过梦境开发脚本修改
2. **依存度/境界**：只能通过脚本计算更新
3. **时间系统**：只能通过TimeSystem修改
4. **真相模式标记**：一旦进入不可逆

### 保护级别

```typescript
enum ProtectionLevel {
  ABSOLUTE,     // 绝对保护：AI永远不能修改

  READONLY,     // 只读：特殊规则控制
```

### 工作流程

```
1. AI生成前
   └─ createDataSnapshot(data)
   └─ 保存所有受保护字段的当前值

2. AI生成后
   └─ validateAndRestoreData(data)
   └─ 对比快照，检测非法修改
   └─ 自动还原被篡改的数据
   └─ 记录篡改日志

3. 写入数据库
   └─ 使用还原后的数据
```

### 代码实现

```typescript
// 创建快照
export function createDataSnapshot(data: SchemaType): DataSnapshot {
  const snapshotData: Record<string, unknown> = {};
  for (const path of Object.keys(PROTECTED_FIELDS)) {
    snapshotData[path] = getNestedValue(data, path);
  }
  return { timestamp: Date.now(), data: snapshotData };
}

// 验证并还原
export function validateAndRestoreData(data: SchemaType): TamperDetectionResult {
  const result = { detected: false, tamperedFields: [], restoredFields: [] };

  for (const [path, level] of Object.entries(PROTECTED_FIELDS)) {
    const originalValue = snapshot.data[path];
    const currentValue = getNestedValue(data, path);

    if (!deepEqual(originalValue, currentValue)) {
      result.detected = true;
      result.tamperedFields.push({ path, level, originalValue, currentValue });

      // 还原数据
      setNestedValue(data, path, originalValue);
      result.restoredFields.push(path);
    }
  }

  return result;
}
```

### 注入位置详解（秦璐代码参考）

> ⚠️ **重要**：以下是经过验证的注入位置和上下文，赵霞游戏必须遵循相同的模式。

#### 代码参考文件

**文件路径**：`src/秦璐/脚本/习惯注入/index.ts`

#### 注入事件钩子

```typescript
// 监听 CHAT_COMPLETION_PROMPT_READY 事件
eventOn(
  tavern_events.CHAT_COMPLETION_PROMPT_READY,
  (event_data: Parameters<ListenerType['chat_completion_prompt_ready']>[0]) => {
    const { chat, dryRun } = event_data;

    // dryRun 时跳过所有状态修改操作（酒馆用于计算 token 的测试运行）
    if (dryRun) {
      console.info(`[习惯注入] dryRun=true，跳过注入`);
      return;
    }

    // ... 注入逻辑
  }
);
```

#### chat 数组结构

`chat` 是一个消息数组，每个消息包含：

- `role`: `'system'` | `'user'` | `'assistant'`
- `content`: 消息内容字符串

#### 注入位置一：状态提示注入（系统消息）

**位置**：第一条 `role='user'` 消息之前

**用途**：状态一致性检查、习惯列表、外观提醒等

```typescript
// 找到第一条 role='user' 的消息位置
let firstUserIndex = -1;
for (let i = 0; i < chat.length; i++) {
  if (chat[i].role === 'user') {
    firstUserIndex = i;
    break;
  }
}

// 插入系统消息
const normalInsertIndex = firstUserIndex === -1 ? chat.length : firstUserIndex;
chat.splice(normalInsertIndex, 0, {
  role: 'system',
  content: statePromptContent,  // 状态提示内容
});
```

**为什么这个位置**：

- 系统消息在用户消息之前，确保 AI 在看到用户输入前就已经了解当前状态
- AI 会根据这些状态信息来调整角色的行为和反应

#### 注入位置二：输入修改注入（替换用户消息）

**位置**：找到最后一条 `role='user'` 消息，直接替换其 `content`

**用途**：场景创建、强制事件触发

```typescript
// 找到最后一条用户消息
let lastUserIndex = -1;
for (let i = chat.length - 1; i >= 0; i--) {
  if (chat[i].role === 'user') {
    lastUserIndex = i;
    break;
  }
}

// 替换用户消息内容
if (lastUserIndex !== -1) {
  chat[lastUserIndex].content = modifiedContent;  // 修改后的内容
}
```

**为什么这个位置**：

- 直接修改用户输入，AI 会认为是用户请求了这个场景
- 比添加新消息更有效，因为 AI 优先响应最近的用户消息

#### 注入位置三：Prefill（引导 AI 开头）

**位置**：chat 数组末尾，添加一条 `role='assistant'` 消息

**用途**：强制 AI 以特定格式开始回复

```typescript
// 添加 Prefill（注意：extended thinking 模式不支持）
if (!currentRequestHasExtendedThinking) {
  chat.push({
    role: 'assistant',
    content: '（门突然被推开，苏文站在门口...）\n\n'
  });
}
```

**注意事项**：

- Claude 的 extended thinking 模式不支持 prefill
- 需要通过 `CHAT_COMPLETION_SETTINGS_READY` 事件检测是否启用了 extended thinking
- 如果启用了，将 prefill 转换为 system 提示作为替代

```typescript
// 检测 extended thinking 模式
eventOn(tavern_events.CHAT_COMPLETION_SETTINGS_READY, (generate_data: any) => {
  currentRequestHasExtendedThinking = generate_data.include_reasoning === true;
});
```

#### 完整注入流程示意

```text
原始 chat 数组:
[
  { role: 'system', content: '角色设定...' },
  { role: 'user', content: '玩家说了XXX' },
  { role: 'assistant', content: 'AI回复了...' },
  { role: 'user', content: '玩家又说了YYY' },  ← 最后一条用户消息
]

注入后 chat 数组:
[
  { role: 'system', content: '角色设定...' },
  { role: 'system', content: '【状态提示】...' },  ← 位置一：状态提示注入
  { role: 'user', content: '玩家说了XXX' },
  { role: 'assistant', content: 'AI回复了...' },
  { role: 'user', content: '【场景切换】...' },   ← 位置二：替换用户消息
  { role: 'assistant', content: '（场景开始...）' }, ← 位置三：Prefill

```

#### 在赵霞游戏中的应用

**场景5触发流程**：

1. 检测用户输入包含安眠药关键词
