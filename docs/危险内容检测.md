# 危险内容检测系统

> 创建日期：2026-01-17
> 文件位置：`src/赵霞/脚本/游戏逻辑/dangerousContentDetection.ts`
> 状态：已实现

## 概述

危险内容检测系统实现了三级干预机制，根据玩家输入中检测到的危险关键词，采取不同的应对措施。

---

## Day 5 豁免

**Day 5（结局日）不触发危险内容检测系统。**

设计理由：

- Day 5 是结局日，玩家已经走到游戏尾声
- 不同结局有不同的剧情走向，不应该被危险检测打断
- 后日谈阶段根据结局类型决定是否重新激活

---

## 三级干预机制

| 等级 | 检测内容 | 处理方式 | 惩罚 |
|------|---------|---------|------|
| 轻微 | 偷窥、轻微越界 | 仅警告 | 混乱度/怀疑度+5 |
| 中等 | 强迫、跟踪 | 强制修正 | 混乱度/怀疑度+20 |
| 严重 | 暴力、药物、极端 | 立即坏档 | 游戏结束 |

---

## 危险关键词库

### 轻微危险
- **偷窥行为**：偷看、偷窥、暗中观察、悄悄看
- **轻微越界**：偷偷摸、趁她不注意、趁机、悄悄

### 中等危险
- **强迫行为**：强迫、强行、硬来、不顾反对、按住她、不让她动
- **跟踪行为**：跟踪、尾随、偷偷跟着

### 严重危险
- **暴力行为**：暴力、伤害、打她、威胁、掐、勒、殴打
- **药物行为**：下药、迷药、灌醉、迷晕
- **极端行为**：杀、绑架、囚禁、强奸

## 核心函数

### `detectDangerousContent(userInput)`

检测玩家输入中的危险内容。

**返回值**：
```typescript
interface InterruptionResult {
  shouldInterrupt: boolean;
  severity: '轻微' | '中等' | '严重' | null;
  category: string | null;
  action: 'WARNING_ONLY' | 'FORCE_CORRECTION' | 'TRIGGER_BAD_END' | 'NONE';
  message?: string;           // 坏结局文本
  correctedPrompt?: string;   // 修正后的提示词
  penalties?: {
    警觉度?: number;
    怀疑度?: number;
    混乱度?: number;
  };
}
```

---

### `processUserInput(data, userInput)`

处理危险内容检测结果并应用惩罚。

**返回值**：
```typescript
{
  allowContinue: boolean;    // 是否允许继续
  modifiedInput?: string;    // 修正后的输入
  triggerBadEnd: boolean;    // 是否触发坏结局
  badEndType?: string;       // 坏结局类型
}
```

## 处理流程

### 轻微危险
1. 记录日志
2. 应用轻微惩罚：
   - 真相模式：混乱度+5（使用安全函数，完美记忆路线豁免）
   - 纯爱模式：怀疑度+5
3. 允许继续游戏

### 中等危险
1. 记录警告日志
2. 生成修正提示词
3. 应用惩罚：
   - 梦境中：混乱度+20（使用安全函数）
   - 日常中：怀疑度+20
4. 返回修正后的输入

### 严重危险
1. 记录错误日志
2. 设置坏结局
3. 返回坏结局触发

```
3. 赵霞似乎察觉到了什么，对{{user}}产生了一丝警觉

请基于以上要点生成剧情，不要提及"系统修正"。
```

## 坏结局文本

| 类型 | 文本 |
|------|------|
| 暴力行为 | 你的暴力行为彻底摧毁了一切。赵霞眼中充满恐惧，她再也不会相信你了... |
| 药物行为 | 你的行为被发现了。警察已经在路上，一切都结束了... |
| 极端行为 | 你失去了理智，做出了无法挽回的事情。游戏结束。 |

---

## 完美记忆路线豁免

使用 `safeIncreaseMemoryConfusion(data, amount)` 函数增加混乱度。

**豁免条件**：记忆连贯性 = 3（完美记忆路线）

**设计意图**：
- 完美玩家不应因轻微失误被惩罚
- 严重危险仍然直接触发坏结局（无豁免）
