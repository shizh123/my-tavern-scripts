# 赵霞项目 - 功能总览

> 创建日期：2026-01-17
> 状态：维护文档

## 项目概述

赵霞项目是一个基于 SillyTavern 酒馆的互动游戏，采用 TypeScript + MVU 框架开发。游戏核心是"时间循环 + 梦境重构"机制，玩家需要在5天内通过进入赵霞的梦境来改变记忆，最终达成不同结局。

## 核心设计理念

1. **双重叙事系统**：纯爱模式（伪装）vs 真相模式（真实）
2. **时间循环**：5天游戏周期，Day 5 判定结局
3. **梦境重构**：5个场景，每个场景需要正确选择才能达成好结局
4. **零BUG时间系统**：单一入口、原子更新、状态一致性

**结局类型说明**：
- **真好结局**：5场景全完成+全正确，记忆连贯性<3
- **完美真爱结局**：真好结局条件 + 记忆连贯性=3（完美记忆路线）
- **假好结局**：5场景全完成但有错误选择
- **普通结局**：时间循环重置（未完成所有场景）
- **坏结局**：怀疑度≥100 或 混乱度≥100 或 危险内容触发
- **混乱结局**：场景5专属坏结局（精神崩溃）

## 源代码文件结构

```
src/赵霞/
├── schema.ts                      # 数据结构定义（Zod）
├── 界面/
│   └── 状态栏/
│       ├── index.ts              # 状态栏入口
│       └── store.ts              # 状态栏数据存储
└── 脚本/
    ├── MVU/
    │   └── index.ts              # MVU 框架入口
    ├── 变量结构/
    │   └── index.ts              # 变量初始化
    ├── 界面/
    │   ├── statusBar.ts          # 状态栏逻辑
    │   └── textMapping.ts        # 文本映射（纯爱/真相模式）
    └── 游戏逻辑/
        ├── index.ts              # 游戏逻辑主入口
        ├── timeSystem.ts         # 时间系统
        ├── stateValidation.ts    # 状态验证
        ├── appearanceSystem.ts   # 双模式外观 + 丈夫位置 + 怀疑度
        ├── dangerousContentDetection.ts  # 危险内容检测
        ├── dreamKeywordDetection.ts      # 梦境关键词 + 部位开发
        ├── dualTrackSystem.ts    # 双轨系统（苦主视角）
        ├── promptInjection.ts    # 动态提示词注入
        ├── dataProtection.ts     # 数据保护（防AI篡改）
        ├── boundaryInterruption.ts # 境界打断系统
        ├── scene5System.ts       # 场景5特殊逻辑
        ├── trueEndingSystem.ts   # 真爱结局
        ├── falseEndingSystem.ts  # 假好结局
        ├── normalEndingSystem.ts # 普通结局
        ├── badEndingSystem.ts    # 坏结局
        └── confusionEndingSystem.ts # 混乱结局
```
